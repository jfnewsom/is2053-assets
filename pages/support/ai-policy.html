AI Policy
ChatGPT, Copilot, and Other AI Tools

AI tools like ChatGPT and GitHub Copilot exist. You’ll probably use them in your career. But in an intro programming course, the goal is to build foundational skills—and you can’t do that if an AI writes your code for you.

Here’s what’s allowed and what isn’t.

These Are Fine
Using AI as a Learning Tool
Debugging Assistance

“Why does this error appear?” — “What does this error message mean?”

Concept Explanation

“What is a while loop?” — “Explain the difference between = and ==”

Syntax Questions

“How do I format an f-string?” — “What’s the syntax for a for loop?”

Understanding Error Messages

“What does ‘IndexError: list index out of range’ mean?”

These Are Violations
Academic Integrity Violations
Generating Code to Submit

“Write a Python program that…” — “Create a function that calculates…”

Having AI Write Functions

Even if you “understand” the code, if AI wrote it, it’s not your work.

Copy/Pasting AI Solutions

This includes “fixing” AI code or using it as a “starting point.”

Posting Assignment Requirements to AI

“Here’s my assignment, help me complete it…”

Scope Compliance
How We Detect AI-Generated Code

CodeGrade uses Scope Compliance checking. It automatically detects if your code uses techniques we haven’t covered yet.

AI-generated code almost always uses advanced features, libraries, or patterns that students in an intro course wouldn’t know yet. When we see list comprehensions in Week 2, or try/except in Week 3, or regex in Week 5… we know.

error
Consequence

Scope violations result in an automatic zero. Using techniques you haven’t learned is clear evidence of outside help.

The Bottom Line
What This Course Is Actually For

Use AI to understand concepts. Don’t use it to write your code.

The goal of this course is to make you a programmer, not to make you good at prompting AI. You’ll have plenty of time to use AI tools later—but first, you need to understand what the AI is actually doing.
