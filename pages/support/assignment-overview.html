Assignment Overview
BUILD → ENHANCE → MASTER

This course uses a three-week rhythm for each module. You don’t start from scratch each week—you build on your own code, adding features as you learn new concepts.

By Week 3, you’re already 60% done. The final assignment is just polishing what you’ve been building all along.

BUILD Week
Week 1 • Foundation
What You Do
  • Read Chapter A (first chapter of the module)
  • Complete Book Exercises for Chapter A
  • Build your foundation program using Chapter A concepts only
  • Save this code—you’ll need it next week!
What’s Due
  • Due Thursday: BookEx X.1
  • Due Tuesday: Lab X.1 (BUILD)
ENHANCE Week
Week 2 • Add Features
What You Do
  • Read Chapter B (second chapter of the module)
  • Complete Book Exercises for Chapter B
  • Copy your Week 1 code, rename to lab-X-2.py
  • Add Chapter B features to your existing code
What’s Due
  • Due Thursday: BookEx X.2
  • Due Tuesday: Lab X.2 (ENHANCE)
MASTER Week
Week 3 • Polish & Complete
What You Do
  • Copy your Week 2 code, rename to lab-X-3.py
  • Add final features and polish
  • Complete all documentation (docstrings, comments)
  • Submit your finished program
What’s Due
  • Due Tuesday: Lab X.3 (MASTER)
  • Module Exam: Opens Sunday, closes Tuesday
Why This Works
The Method Behind the Madness
No Last-Minute Panic
You code every week, not just at the end. The work is spread out so you’re never cramming.
Build on Your Own Code
You’re not starting over each assignment. Your Week 1 code becomes Week 2, which becomes Week 3.
Catch Problems Early
If something’s wrong in Week 1, you’ll find out before it breaks everything in Week 3.
Real-World Skills
Professional developers iterate on code all the time. This is how software actually gets built.
