Assignment Overview
BUILD → ENHANCE → MASTER

This course uses a three-unit rhythm for each module. You don’t start from scratch each unit—you build on your own code, adding features as you learn new concepts.

By Unit 3, you’re already 60% done. The final assignment is just polishing what you’ve been building all along.

BUILD Unit
Unit 1 • Foundation
What You Do
  • Read Chapter A (first chapter of the module)
  • Complete Book Exercises for Chapter A
  • Build your foundation program using Chapter A concepts only
  • Save this code—you’ll need it next unit!
What’s Due
  • BookEx X.1 — due before the lab
  • Lab X.1 (BUILD) — due at end of unit
ENHANCE Unit
Unit 2 • Add Features
What You Do
  • Read Chapter B (second chapter of the module)
  • Complete Book Exercises for Chapter B
  • Copy your Unit 1 code, rename to lab-X-2.py
  • Add Chapter B features to your existing code
What’s Due
  • BookEx X.2 — due before the lab
  • Lab X.2 (ENHANCE) — due at end of unit
MASTER Unit
Unit 3 • Polish & Complete
What You Do
  • Copy your Unit 2 code, rename to lab-X-3.py
  • Add final features and polish
  • Complete all documentation (docstrings, comments)
  • Submit your finished program
What’s Due
  • Lab X.3 (MASTER) — due at end of unit
  • Module Exam — check the course schedule for exam window
Why This Works
The Method Behind the Madness
No Last-Minute Panic
You code every unit, not just at the end. The work is spread out so you’re never cramming.
Build on Your Own Code
You’re not starting over each assignment. Your Unit 1 code becomes Unit 2, which becomes Unit 3.
Catch Problems Early
If something’s wrong in Unit 1, you’ll find out before it breaks everything in Unit 3.
Real-World Skills
Professional developers iterate on code all the time. This is how software actually gets built.
