Section 2: Input and Output

If you did the labs, you’ve used all of these. Can you explain how each works?

The input() Function
  • input() — displays a prompt, waits for user, returns their response
  • Return type — input() ALWAYS returns a string, even if user types a number
  • Storing input — name = input("Enter name: ")
  • Prompt string — the message shown to the user
Key question: If the user types 42 in response to input(), what data type is returned?
Type Conversion
  • int() — converts a string to an integer
  • float() — converts a string to a floating-point number
  • str() — converts a number to a string
  • Chained conversion — age = int(input("Enter age: "))
  • ValueError — what happens if you try int("hello")
The print() Function
  • print() — displays output to the screen
  • Multiple arguments — print(a, b, c) separates with spaces
  • end parameter — print(x, end="") suppresses the newline
  • sep parameter — print(a, b, sep="-") changes the separator
F-Strings
  • f-string syntax — f"text {variable} more text"
  • Expression in braces — f"Total: {price * quantity}"
  • Format specifier — f"{value:.2f}" for 2 decimal places
  • Comma formatting — f"{big_number:,}" adds thousands separators
  • Percentage — f"{rate:.1%}" displays as percentage

Order of Operations (PEMDAS)
  • Parentheses — highest priority, evaluated first
  • Exponents — ** evaluated next
  • Multiplication/Division/Modulo — *, /, //, % left to right
  • Addition/Subtraction — +, - left to right, lowest priority

What is 2 + 3 * 4?      What is (2 + 3) * 4?      What is 10 - 4 - 2?  

Mixed Type Arithmetic
  • int + int = int — no surprises
  • float + anything = float — floats are “contagious”
  • int / int = float — true division ALWAYS returns float

Section 4: Decision Structures

Python uses indentation to define blocks. Get this wrong, and nothing works.

The if Statement
  • if statement — executes code block only when condition is True
  • Condition — an expression that evaluates to True or False
  • Colon — required at the end of the if line
  • Indentation — the indented block IS the code that runs conditionally
if-elif-else Chain
  • elif — short for “else if”, tests another condition
  • Multiple elif — can chain as many as needed
  • First match wins — only the FIRST true condition’s block executes
  • Optional else — catches everything that didn’t match above
Key question: In an if-elif-elif-else chain, how many blocks can execute?

Section 7: Professional Practices

This section goes beyond the exam. These practices will make you a better programmer.

PEP 8 Highlights
  • Line length — maximum 79 characters per line
  • Indentation — use 4 spaces (not tabs)
  • Naming — lowercase_with_underscores for variables and functions
  • Constants — ALL_CAPS_WITH_UNDERSCORES
  • Whitespace around operators — x = 5 not x=5
Common Beginner Mistakes
  • = vs == — assignment vs comparison (most common error)
  • Forgetting the colon — if x > 5: needs the colon
  • input() returns string — forgetting to convert with int() or float()
  • Case sensitivity — Score and score are different variables
  • String vs variable — print("name") vs print(name)
