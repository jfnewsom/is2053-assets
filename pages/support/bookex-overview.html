Book Exercises
Practice Problems to Prepare You for Labs

Book Exercises (BookEx) are short practice problems from your Gaddis textbook. They’re designed to help you master individual concepts before applying them in your lab assignments.

Think of them as warm-ups before the big game. Each BookEx isolates a specific skill—variables, loops, functions—so you can practice without the complexity of a full project.

lightbulb
Key Insight

When you’re stuck on a lab, go back to the relevant BookEx. The patterns you practiced there are the same patterns you need in your lab!

The BookEx Workflow

How to Use These Exercises

  1. Read the assigned textbook chapter
  2. Open the BookEx assignment sheet for that chapter
  3. Complete the exercises (usually 3–5 short programs)
  4. Submit your solutions to CodeGrade
  5. Use the patterns you learned in your lab assignment!

How BookEx Are Scored

Binary Grading — 100 or 0

BookEx grading is binary on a single criterion: does your code produce the correct output? If yes — 100. If no, or you didn’t submit — 0. No partial credit. No style points. No rubric to navigate. The point of these exercises is to put working code in front of you that you can lift directly into your labs — so the only thing that matters is whether the working code is, in fact, working.

troubleshoot
The Real Lesson

You’re copying short programs straight from your Gaddis textbook into VS Code. If your typed-out version runs and the output matches, you’re done. But you’ll make typos, and the program won’t run, and you’ll have to find what’s wrong and fix it. That’s the lesson. Reading the error, locating the typo, making it run — that’s the skill every coding assignment after this one requires.

You get unlimited submissions before the deadline, same as every coding assignment. There’s no reason anyone shouldn’t have a 100 on every BookEx. It’s up to you.
