Section 1: File Basics

Files let your program save data that persists after the program ends. Get the mode wrong, and you lose data.

File Concepts
  • File — a named location on disk that stores data
  • Text file — stores data as human-readable characters
  • Sequential access — reading/writing from beginning to end, in order
  • File object (file handle) — the connection between your program and the file
File Modes
ModePurposeBehavior if File Exists
'r'Read onlyOpens for reading
'w'Write (create/overwrite)ERASES existing content!
'a'Append (add to end)Adds to existing content
Key question: What happens to existing data when you open a file with 'w' mode?
Closing Files
  • close() method — closes the file
  • Why close? — releases system resources, ensures data is saved
  • with statement — automatically closes file when block ends (preferred)
Section 2: Reading and Writing Files

Know the difference between read(), readline(), and iterating. Each has its purpose.

Writing to Files
  • write() method — writes a string to the file
  • No automatic newline — write() doesn’t add \n; you must include it
  • Writing numbers — must convert to string first: file.write(str(42))
Reading from Files
  • read() — reads entire file as one string
  • readline() — reads one line (including the \n)
  • readlines() — reads all lines into a list
  • Iterating — for line in file: reads one line at a time (memory efficient)
The Newline Issue
  • \n at end of lines — readline() includes it; strip() removes it
  • Empty string — readline() returns '' at end of file
Key question: Why does readline() return 'Austin\n' instead of just 'Austin'?
Section 3: Exception Handling

Exceptions let you handle errors gracefully instead of crashing. Try, except, recover.

Exception Concepts
  • Exception — an error that occurs during program execution
  • Handling an exception — catching and responding to an error
  • Crash — what happens if an exception isn’t handled
  • try block — code that might raise an exception
  • except block — code that handles the exception
Common Exceptions
ExceptionWhen It Occurs
FileNotFoundErrorOpening a file that doesn’t exist (in 'r' mode)
ValueErrorConverting invalid string to number: int('abc')
ZeroDivisionErrorDividing by zero: 10 / 0
IndexErrorAccessing list index that doesn’t exist
Key question: What’s the difference between preventing an error and handling an exception?
Section 4: List Basics

Lists store multiple values in a single variable. They’re mutable—you can change them.

List Fundamentals
  • List — an ordered, mutable collection of items
  • Element — a single item in the list
  • Index — the position of an element (starts at 0)
  • Length — number of elements: len(my_list)
  • Empty list — a list with no elements: []
Accessing Elements
  • Positive index — my_list[0] is first element
  • Negative index — my_list[-1] is last element
  • IndexError — accessing an index that doesn’t exist

Section 5: List Methods and Operations

Lists have methods that modify them in place. Know what each returns.

Adding Elements
  • append(item) — adds item to the END of the list
  • insert(index, item) — inserts item at the specified index
  • Both modify the list in place — they don’t return a new list
Removing Elements
  • remove(item) — removes FIRST occurrence of item (ValueError if not found)
  • pop() — removes and returns the LAST element
  • pop(index) — removes and returns element at index
  • del my_list[index] — deletes element at index
Other Useful Operations
  • in operator — 'Austin' in cities returns True/False
  • len(), sum(), min(), max()
  • sort() — sorts list in place; reverse() — reverses in place
Key question: What’s the difference between remove() and pop()?

Section 7: Professional Practices

File handling and data structure skills are career-critical.

File Handling Best Practices
  • Always use with statement — guarantees file closes even if error occurs
  • Handle FileNotFoundError — don’t assume file exists
  • Be careful with 'w' mode — it destroys existing data
Common Module 3 Mistakes
  • Opening file in wrong mode — reading with 'w' erases it!
  • Forgetting newlines — write() doesn’t add them automatically
  • Not stripping newlines — readline() includes \n
  • Confusing append() return value — it returns None, not the list
