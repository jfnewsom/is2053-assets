Python Code Style Checker

Flake8 is a code style checker that runs automatically when you submit to CodeGrade. It checks your Python code against PEP 8—the official Python style guide used by professional developers worldwide.

These aren't just suggestions—they're part of your Code Style grade (10%). Clean code is professional code!

Fix These First! (-15 Points Each)
F401: Module Imported But Unused

You imported something but never used it. Remove the import!

❌ PROBLEM
import math
# Never used math!
print("Hello")
✓ FIX
# Removed unused import
print("Hello")
F841: Variable Assigned But Never Used

You created a variable but never did anything with it.

❌ PROBLEM
result = 5 + 3
# Never printed or used!
✓ FIX
result = 5 + 3
print(result)
Common PEP 8 Issues (-5 Points Each)
E302: Expected 2 Blank Lines

Put 2 blank lines before every function definition.

E303: Too Many Blank Lines

Never have more than 2 consecutive blank lines.

E501: Line Too Long

Keep lines under 79 characters. Break long strings or calculations across multiple lines.

W291/W293: Trailing Whitespace

Remove invisible spaces at the end of lines. VS Code can do this automatically!

E111/E117: Indentation Errors

Use exactly 4 spaces per indentation level. Never mix tabs and spaces!

Check Before Submitting

Before submitting, check these common issues:

  • ☐ 2 blank lines before each function
  • ☐ 4 spaces for indentation (not tabs)
  • ☐ Lines under 79 characters
  • ☐ No trailing whitespace
  • ☐ Space after commas: func(a, b, c)
  • ☐ Spaces around operators: x = 5 + 3
  • ☐ All variables you create are actually used
  • ☐ All imports at the top are actually needed
