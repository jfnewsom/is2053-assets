Flake8 Guide
Python Code Style Checker

Flake8 is a code style checker that runs automatically when you submit to CodeGrade. It checks your Python code against PEP 8—the official Python style guide used by professional developers worldwide.

These aren’t just suggestions—they’re part of your Code Style grade (10%). Clean code is professional code!

Reading Errors
Understanding Error Messages

Flake8 errors follow this format:

lab-1-1.py:12:5: E302 expected 2 blank lines, found 1
Filename
Which file the error is in

Which line to look at

Position on that line
Error Code
The type of problem found
Error Code Prefixes
Prefix Category
E Style errors (PEP 8 violations)
W Style warnings (minor issues)
F PyFlakes errors (code problems) — most serious!
Critical Errors
Fix These First! (−15 Points Each)
F401 — Module Imported But Unused

You imported something but never used it. Remove the import!

close
Problem
import math
# math is never used!
print("Hello")
check
Fix
# Removed unused import
print("Hello")
F841 — Variable Assigned But Never Used

You created a variable but never did anything with it.

close
Problem
result = 5 + 3
# result is never printed or used!
check
Fix
result = 5 + 3
print(result)
Style Errors
Common PEP 8 Issues (−5 Points Each)
Code Problem Fix
E302 Expected 2 blank lines before function Put 2 blank lines before every function definition
E303 Too many blank lines Never have more than 2 consecutive blank lines
E501 Line too long Keep lines under 79 characters; break long strings across multiple lines
W291 / W293 Trailing whitespace Remove invisible spaces at the end of lines (VS Code can do this automatically)
E111 / E117 Indentation errors Use exactly 4 spaces per level — never mix tabs and spaces
E225 Missing spaces around operator Write x = 5 + 3 not x=5+3
E231 Missing space after comma Write func(a, b, c) not func(a,b,c)
Quick Fixes
Check Before Submitting

Run through this list before every submission:

  • 2 blank lines before each function definition
  • 4 spaces for each indentation level (not tabs)
  • All lines under 79 characters
  • No trailing whitespace at the end of lines
  • Spaces after commas: func(a, b, c)
  • Spaces around operators: x = 5 + 3
  • All variables you created are actually used
  • All imports at the top are actually needed
