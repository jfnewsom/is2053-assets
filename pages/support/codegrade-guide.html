Your Code Submission & Feedback System
What Is CodeGrade

CodeGrade is where you submit your Python assignments and get instant feedback. It automatically tests your code against expected outputs and checks for code quality issues.

lightbulb
Unlimited Submissions

You get unlimited submissions before the deadline. Submit early and often to see what’s working and what needs fixing.

How to Submit

Submission is a five-step loop you’ll run as many times as you need before the deadline:

  1. Open the assignment in Canvas and click the CodeGrade link
  2. Upload your .py file (make sure the filename matches exactly!)
  3. Wait for tests to run — usually takes 10–30 seconds
  4. Review your results — see what passed and what failed
  5. Fix issues and resubmit — you have unlimited attempts!
error
Filename Matters

Your file must be named exactly as specified (e.g., lab-1-1.py). Wrong filename = tests won’t run.

Reading Feedback

After submission, you’ll see five categories of feedback. Each category looks at a different aspect of your code:

Scope Compliance

Did you use techniques we haven’t covered yet? Using advanced features that are out of scope results in an automatic zero on that test.

Correctness Tests

Does your output match what’s expected? These tests run your code with specific inputs and check the output character by character — both visible test cases (you can see) and hidden test cases (different values to prevent hardcoding).

Required Techniques

Did you use the required programming constructs? For example: “must use a while loop,” “must use a named constant,” etc. CodeGrade checks for the specific concepts taught in the module’s chapters.

Code Style (Flake8)

Is your code properly formatted? Line length, spacing, naming conventions, and more. See the Flake8 Guide for a full breakdown of every error code.

Documentation

Does your file have the required comments? CodeGrade checks for a module docstring at the top of the file (with your name, UTSA ID, section, and description) and docstrings for all functions.

For point values per category and how they factor into your final grade, see the Rubrics section on Grading Info.

Common Issues

If CodeGrade is giving you trouble, scan this table first — most issues fall into one of these buckets:

Error What It Means & How to Fix It
No tests ran Your filename is wrong. Check that it matches exactly — case-sensitive, hyphens not underscores (e.g., lab-1-1.py).
Output doesn’t match Check for extra spaces, missing punctuation, or wrong capitalization. Output must match exactly, character by character.
Scope violation You used a technique we haven’t covered yet. Check the assignment sheet for the list of permitted techniques.
Flake8 errors Code style issues. Common causes: lines over 79 characters, missing spaces around operators, wrong indentation. See the Flake8 Guide.
Program timed out Your code has an infinite loop. Check your while loop conditions and make sure the loop variable is being updated.
