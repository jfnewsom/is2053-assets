Section 1: The while Loop

The while loop repeats as long as its condition remains True. Get the condition wrong, and you get an infinite loop.

while Loop Basics
  • while loop — repeats a block of code while a condition is True
  • Loop condition — the boolean expression tested before each iteration
  • Iteration — one complete execution of the loop body
  • Sentinel value — a special value that signals the loop should end
Loop Control
  • Loop variable — a variable that changes each iteration, eventually making the condition False
  • Priming read — getting initial input before the loop starts
  • Update statement — code that modifies the loop variable (usually inside the loop)
  • Infinite loop — a loop that never terminates; Ctrl+C to stop
Key question: What three things must be true for a while loop to eventually terminate?
Input Validation Pattern
  • Input validation — checking that user input is acceptable before using it
  • Pattern: get input → while invalid: display error, get input again
Section 2: The for Loop and range()

When you know how many times to repeat, use for. It handles the counting for you.

for Loop Basics
  • for loop — iterates over a sequence of values
  • Count-controlled loop — loop that executes a specific number of times
  • Target variable — the variable that takes each value in the sequence
The range() Function
  • range(stop) — generates 0, 1, 2, … up to but NOT including stop
  • range(start, stop) — generates start, start+1, … up to but NOT including stop
  • range(start, stop, step) — generates values with specified step
  • Off-by-one — range(5) gives 0–4, not 0–5
range() CallValues Generated

range(5, 5)
Nested Loops
  • Nested loop — a loop inside another loop
  • Inner loop — completes ALL its iterations for EACH iteration of outer loop
  • Total iterations — outer count × inner count
Key question: If outer loop runs 3 times and inner loop runs 4 times, how many times does the inner body execute?
Section 3: Accumulators

Accumulators build up a result across multiple iterations. Initialize, update, use.

The Accumulator Pattern
  • Accumulator — a variable that accumulates a value across loop iterations
  • Running total — total += value
  • Counter — count += 1
  • Initialize before loop — accumulator must have starting value (usually 0)
  • Update inside loop — add/multiply/etc. each iteration

Section 4: Functions

Functions let you write code once and use it many times. They make programs modular and maintainable.

Function Basics
  • Function — a named block of code that performs a specific task
  • def keyword — starts a function definition
  • Function header — the def line with function name and parameters
  • Function body — the indented code inside the function
  • Function call — executing a function by using its name with parentheses
The main() Function Pattern
  • main() function — convention for the program’s starting point
  • if __name__ == "__main__": — ensures main() only runs when file is executed directly
Key question: What’s the difference between defining a function and calling it?
Section 5: Parameters and Return Values

Parameters let you pass data INTO functions. Return values let you get data OUT.

Parameters and Arguments
  • Parameter — variable in the function definition that receives a value
  • Argument — actual value passed to the function when calling it
  • Positional argument — matched to parameter by position
  • Keyword argument — matched by name: func(name="Bob")

Return Values
  • return statement — sends a value back to the caller
  • Capturing return value — result = calculate(5)
  • None — what a function returns if it has no return statement
  • Void function — a function that doesn’t return a value (returns None)
Local Variables and Scope
  • Local variable — variable created inside a function; exists only there
  • Scope — where a variable can be accessed
  • Local scope — inside the function where variable was created
  • Global scope — outside all functions (avoid using global variables!)
Key question: Why can’t code outside a function access variables created inside that function?

Section 7: Professional Practices

These practices separate hobbyists from professionals.

Function Design Principles
  • Single responsibility — each function should do ONE thing well
  • Descriptive names — calculate_tax() not calc() or doIt()
  • Reasonable length — if function is >20 lines, consider splitting
  • Avoid global variables — pass data through parameters instead
Common Module 2 Mistakes
  • Infinite loop — forgetting to update the loop variable in while
  • Off-by-one errors — range(5) gives 0–4, not 1–5
  • Forgetting to initialize accumulator — total must be 0 before loop
  • Confusing parameter and argument — parameter in def, argument in call
  • Forgetting to capture return value — result = func() not just func()
