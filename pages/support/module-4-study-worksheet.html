Section 1: String Indexing and Slicing

Strings are sequences. Access individual characters by index, or extract substrings with slicing.

String Indexing
  • Index — position of a character (starts at 0)
  • Positive index — counts from the beginning: s[0] is first character
  • Negative index — counts from the end: s[-1] is last character
  • Immutable — strings cannot be changed; s[0] = 'X' causes an error

String Slicing
  • Slice — extracting a substring: s[start:end]
  • Start index — where slice begins (included)
  • End index — where slice stops (NOT included)
  • Default start — omitting start means 0: s[:3] = s[0:3]
  • Default end — omitting end means end of string
  • Step — optional third value: s[start:end:step]

Section 2: String Methods

String methods return NEW strings. The original is unchanged (strings are immutable).

Case and Whitespace Methods
  • upper() — returns string in all uppercase
  • lower() — returns string in all lowercase
  • capitalize() — first character uppercase, rest lowercase
  • strip() — removes leading and trailing whitespace
  • lstrip() / rstrip() — removes leading/trailing only
Search, Replace, and Testing
  • find(sub) — returns index of first occurrence, or -1 if not found
  • replace(old, new) — returns new string with all occurrences replaced
  • isdigit() — True if all characters are digits
  • isalpha() — True if all characters are letters
  • startswith(prefix) / endswith(suffix)
Split and Join
  • split() — splits string into list of words (whitespace by default)
  • split(sep) — splits on specified separator
  • join(list) — joins list elements with string as separator
Key question: Why do string methods return new strings instead of modifying the original?
Section 3: Dictionary Basics

Dictionaries store key-value pairs. Fast lookup by key, but keys must be unique.

Dictionary Concepts
  • Dictionary — collection of key-value pairs
  • Key — unique identifier used to access a value
  • Value — data associated with a key
  • Mutable — dictionaries can be changed after creation
  • Literal — cities = {'Austin': 165, 'Houston': 197}
  • Empty dictionary — my_dict = {} or my_dict = dict()
Accessing and Modifying
  • Bracket notation — my_dict['key'] returns the value
  • KeyError — accessing a key that doesn’t exist
  • in operator — 'Austin' in my_dict checks if key exists
  • Adding — my_dict['new_key'] = value
  • Deleting — del my_dict['key']
Key question: Why must dictionary keys be immutable (like strings or numbers)?
Section 4: Dictionary Methods

Dictionary methods give you safe ways to access and iterate. Use get() to avoid KeyError.

Safe Access with get()
  • get(key) — returns value if key exists, None if not
  • get(key, default) — returns value if key exists, default if not
  • Why use get()? — avoids KeyError, no need for try/except

Iteration Methods
  • keys() — returns all keys
  • values() — returns all values
  • items() — returns key-value pairs as tuples
  • Shortcut — for key in my_dict: iterates over keys
Section 5: Sets

Sets store unique values only. Duplicates are automatically removed. No indexing.

Set Concepts
  • Set — unordered collection of unique elements
  • Unique — duplicates are automatically eliminated
  • Unordered — no indexing, no guaranteed order
  • Literal — my_set = {1, 2, 3}
  • From list — my_set = set([1, 2, 2, 3]) gives {1, 2, 3}
  • Empty set — my_set = set() (NOT {} — that’s an empty dict!)
Set Operations
  • add(element), remove(element), discard(element)

Section 7: Professional Practices

Data structure choice is a key engineering decision.

Choosing the Right Data Structure
  • List — when order matters, when you need duplicates, when you access by position
  • Dictionary — when you need fast lookup by key, when pairing related data
  • Set — when you need uniqueness, when checking membership frequently
Common Module 4 Mistakes
  • Off-by-one in slicing — s[0:3] gets 3 characters, not 4
  • Trying to modify strings — strings are immutable; use s = s.upper()
  • KeyError with dictionaries — use get() or check with in first
  • Empty set confusion — {} is dict, use set() for empty set
  • Forgetting isdigit() checks decimals — '12.5'.isdigit() is False
