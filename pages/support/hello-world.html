Step 1 — Open Folder, Not File (Your Day-One Workflow)

Right now you have two ways to get hello.py in front of VS Code. Pay attention to which one we’re using, because this is the workflow rule every assignment in this course depends on.

  • The wrong way — double-click hello.py in Finder/Explorer. VS Code opens with just that one file, disconnected from any folder. Python doesn’t know what directory to look in for data files. You won’t notice the problem today (there are no data files yet), but starting in Module 4 you’ll get FileNotFoundError messages that make no sense.
  • The right way — open the folder in VS Code first, then click the file from the sidebar. Python automatically uses that folder as its working directory. Your data files just work. This is how every professional Python developer works.

We’re going to do it the right way every single time, starting now. Even though hello.py has no data files and you could get away with the wrong way today, building the habit on day one means you never trip over the problem later.

To open the folder:

  1. In Finder/Explorer, navigate to is2053/start_here/
  2. Drag the start_here folder onto the VS Code icon — OR — open VS Code and use File → Open Folder, then pick start_here
  3. You should now see hello.py listed in the Explorer sidebar on the left. This means VS Code has the folder open as a workspace, and the file is accessible from inside it. That’s the workflow.
Step 2 — Write the program and run it locally

Now that the folder is open in VS Code, click hello.py in the Explorer sidebar on the left. It opens in the editor. Type exactly this line: 

print("Hello, World!")
match_case
Match the Cases and Punctuation Exactly

CodeGrade is going to compare what your program prints against what it expects, character by character. It is case-sensitive and punctuation-sensitive.

Type the line exactly as shown above:

  • Capital H in Hello, capital W in World
  • A comma after Hello, followed by a single space
  • An exclamation point at the end
  • Double quotes around the whole string

If you type print("hello world") or print("Hello world!") or even print("Hello,World!") (missing the space), the test will fail. Get used to this now: every assignment in this course is graded by a computer that does not interpret — only compare. Exact matters.

The easiest way to nail this: copy the spec, don’t retype it. When the instructions show you the exact text the program should print, select it, copy it, paste it into your code. Zero typos, zero case mistakes, zero missed punctuation. Professionals do this constantly — nobody manually retypes a string when they can copy it from the spec. Save your typing energy for the parts of the program that actually require thinking.

If the Play Button Doesn’t Work

The most common cause: Python wasn’t added to PATH during install. If you see “Python is not recognized” or a similar error, here’s the fix:

On Windows: Re-run the Python installer from python.org and this time check the “Add Python to PATH” box at the bottom of the first installer screen before you click Install. It takes 2 minutes and is the simplest fix. If you already know how to edit the PATH environment variable in Windows, you can do that instead. If you just said “what is the PATH?” — just reinstall.

On Mac: VS Code probably hasn’t picked which Python to use yet. Open the Command Palette with Cmd + Shift + P, type Python: Select Interpreter, and pick the one with the highest version number (should be 3.14 or newer).

After either fix, close VS Code completely, reopen it, open the start_here folder again, and try the Play button. Still stuck? Drop a note in Discord #help-python or book a 1-on-1.
