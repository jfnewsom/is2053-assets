Hello World
Your First Submission · 15–20 minutes

First time visiting this page outside Start Here? Run through the full Start Here checklist before you tackle this one — Python install, VS Code setup, AI autocomplete off, workspace folder. This page assumes all of that is done.

You’re going to do a complete end-to-end workflow for the first time: open a folder in VS Code, write a Python program, run it locally, then submit it to CodeGrade (the autograder we use for every coding assignment in this course). This submission is worth zero points — its only job is to make sure your setup works and to walk you through CodeGrade signup before there’s a grade on the line.

Step 1 — Open the start_here folder in VS Code

In your starter folder, find is2053/start_here/. Inside it is an empty file called hello.py. Open the folder, not the file — this is the workflow rule we’ll use for every assignment in this course:

  1. In Finder/Explorer, navigate to is2053/start_here/
  2. Drag the start_here folder onto the VS Code icon — OR — open VS Code and use File → Open Folder, then pick start_here
  3. You should now see HELLO.PY in the Explorer sidebar on the left
Step 2 — Write the program and run it locally

Click hello.py in the sidebar to open it. Type exactly this line: 

print("Hello, World!")

Save the file (Cmd + S on Mac, Ctrl + S on Windows). Then click the Play button in the upper-right corner of the editor. A terminal panel opens at the bottom of VS Code and you should see: 

Hello, World!

That’s your first Python program working locally. Now we send it to CodeGrade.

error
If the Play Button Doesn’t Work

The most common cause is Python wasn’t added to your PATH during install. If you see “Python is not recognized” or a similar error, head to Discord or grab a 1-on-1 via Calendly — we’ll get it sorted before you go further. Don’t skip ahead with a broken setup; everything from here on depends on it.

Step 3 — Submit to CodeGrade through Canvas

All assignments in this course are submitted through Canvas, but the actual grading runs in CodeGrade. The first time you submit, Canvas walks you through CodeGrade signup automatically — one-time setup, then every future submission goes straight through.

rocket_launch Launch Hello World in Canvas

  1. Open Canvas and go to the Modules section
  2. Find the “Hello World” assignment in Module 0 (Start Here)
  3. Click the assignment, then click Load Hello World in a new window
  4. A CodeGrade window opens. If this is your first time, it asks you to create an account using your UTSA email
  5. CodeGrade asks for payment — see the callout below for your options
  6. Once you’re paid up, you see the submission screen. Drag hello.py from your VS Code Explorer sidebar (or your Finder/Explorer) into the upload area
  7. Click Submit
payment
Payment Options

CodeGrade access is a one-time fee for the semester. You have two choices:

  • Credit card — pay CodeGrade directly during signup. Fastest option, takes about 2 minutes.
  • Activation code from the bookstore — purchase one from the UT San Antonio Bookstore, then enter the code when CodeGrade prompts you. Use this if you’re paying with financial aid.
Step 4 — Confirm it worked

After submitting, CodeGrade processes your code and shows you a feedback panel. You should see:

  • A green checkmark or “passed” indicator on your submission
  • Your code displayed back to you in the right panel
  • A score of 0/0 (this assignment is zero points by design)

That’s the complete end-to-end workflow. From now on, every coding assignment in this course follows this exact pattern: open the assignment folder, write code, test locally, submit through Canvas. The signup pain is now behind you.

Need more detail on reading CodeGrade feedback later in the term? See the CodeGrade Guide for a complete walkthrough of correctness tests, scope compliance, and Flake8 style checks.

When you’re done, head back to Start Here to finish the remaining checkpoints, or jump to Module 1 for your first real lab.
