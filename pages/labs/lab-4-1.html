Load City Distances

Parse the city_distances.txt file and build a nested dictionary structure.

A function that reads city_distances.txt and creates a nested dictionary where you can look up distances like distances['San Antonio']['Houston'] → 200

In Modules 1-3, you had 20+ constants like SAT_TO_HOUSTON = 200. Adding a new city meant adding 5+ new constants AND updating multiple if-elif chains. With a nested dictionary loaded from a file, adding a city is just adding lines to the data file. Your code doesn't change at all!

"When testing file-loaded data, I always check three things: Does the file load correctly? Is the data structured right? What happens if the file is malformed? Print your dictionary right after loading it—if the keys and values look right, your parsing worked. If not, you found the bug early."

QA Lead, Bat City Collective

Program 8-10 (split_date.py): You used .split('/') to break a date string into parts. Now use .split(',') to break each CSV line into city1, city2, and distance!

Program 9-1 (card_dealer.py): You created a dictionary with key-value pairs. Now you're building a nested dictionary where each city maps to another dictionary of destinations.

  1. Create a new file called lab-4-1.py with your standard docstring
  2. Add imports: import random and import function_library
  3. Define load_city_distances(filename) that takes a filename parameter
  4. Create an empty dictionary: distances = {}
  5. Open the file and loop through each line
  6. For each line: use line.strip().split(',') to get parts
  7. Extract: city1 = parts[0], city2 = parts[1], distance = int(parts[2])
  8. Check if city1 is already a key; if not, create an empty dictionary for it
  9. Add the destination: distances[city1][city2] = distance
  10. Close the file and return the distances dictionary

KeyError when adding to nested dict Check if city1 exists first: if city1 not in distances:
Forgetting to convert distance to int Use int(parts[2]) — split() returns strings!

Load City Info

Parse the city_info.txt file and build a dictionary with nested city data.

A function that reads city_info.txt and creates a dictionary where each city maps to another dictionary containing its type and tagline. Like this: city_info['Houston']['tagline'] → 'Space City'

Program 9-1 (card_dealer.py): You built a dictionary where card names map to values. Now build a dictionary where city names map to dictionaries of attributes!

  1. Define load_city_info(filename) that takes a filename parameter
  2. Create an empty dictionary: info = {}
  3. Open the file and loop through each line
  4. For each line: use line.strip().split(',') to get parts
  5. Extract: city = parts[0], city_type = parts[1], tagline = parts[2]
  6. Create nested dict: info[city] = {'type': city_type, 'tagline': tagline}
  7. Close the file and return the info dictionary

Data-Driven Display

Create helper functions that use dictionaries to display destinations and look up information.

Functions that display available destinations from a city and help with navigation—all driven by your dictionaries, not hardcoded values!

  1. Create display_destinations(city, distances) that takes current city and distances dict
  2. Loop through destinations: use a counter variable to number options (1, 2, 3...)
  3. Print each: option number, destination name, and distance
  4. Create get_destination_by_number(city, distances, choice)
  5. Convert dict keys to list: destinations = list(distances[city].keys())
  6. Return the destination at index choice - 1 (user picks 1, but index is 0)
  7. Create is_win_city(city, city_info) that checks if type is 'WIN'
  8. Create is_dead_end(city, city_info) that checks if type is 'DEAD_END' or 'WRONG_WAY'
