Copy & Load Hazards

Start from your Lab 4.1 code and add a new file-loading function for hazards.

A function that reads hazards.txt and returns a list of dictionaries. Each hazard dictionary has keys for name, heads message, and tails message.

This pattern—loading structured data from files into a list of dictionaries—is everywhere in professional development. JSON APIs return lists of objects, database queries return lists of records, and configuration files store lists of settings. Master this pattern and you've unlocked how real applications manage data.

Your load_city_info() from 4.1 already creates dictionaries from CSV lines. This is the same pattern, but you'll append each hazard dict to a list instead of adding to a parent dict.

  1. Copy your lab-4-1.py and rename it to lab-4-2.py
  2. Update your docstring to say "Lab 4.2: Hazards on the Highway"
  3. Define load_hazards(filename) that takes a filename parameter
  4. Create an empty list: hazards = []
  5. Open the file and loop through each line
  6. For each line: use line.strip().split(',') to get parts
  7. Create a hazard dict: {'name': parts[0], 'heads': parts[1], 'tails': parts[2]}
  8. Append to list: hazards.append(hazard)
  9. Close the file and return the hazards list

D20 Travel System

Replace instant travel with dice-roll-based progression and random hazard encounters.

A travel_to() function where you roll a D20 each turn, travel (roll 10) miles, and have a 30% chance of encountering a hazard. The journey continues until you've covered the full distance.

  1. Define travel_to(destination, distance, hazards)
  2. Initialize: remaining = distance, miles_traveled = 0, hazards_faced = 0, hazards_escaped = 0
  3. Create while loop: while remaining > 0:
  4. Roll D20: roll = function_library.roll_d20()
  5. Calculate miles: miles = roll * 10
  6. Cap miles if it would overshoot: if miles > remaining: miles = remaining
  7. Update accumulators: miles_traveled, remaining
  8. Print progress: show roll, miles gained, remaining
  9. Check for hazard (30% chance, only if not done): function_library.check_percentage(30)
  10. If hazard triggered: call handle_hazard(), update counters
  11. Return tuple: (miles_traveled, hazards_faced, hazards_escaped)

Only check for hazards when remaining > 0. You don't want a hazard on your last turn when you've already arrived!
