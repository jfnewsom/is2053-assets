Copy & Load Hazards
Start from your Lab 4.1 code and add a new file-loading function for hazards.
What You're Building

A function that reads hazards.txt and returns a list of dictionaries. Each hazard dictionary has keys for name, heads message, and tails message.

Why This Matters

Loading structured data from files into a list of dictionaries is everywhere in professional development. JSON APIs return lists of objects, database queries return lists of records, and configuration files store lists of settings. Master this pattern and you've unlocked how real applications manage data.

BookEx Patterns
  • Program 9-1 — card_dealer.py — Your load_city_info() from Lab 4.1 already creates dictionaries from CSV lines. This is the same pattern — but you'll append each hazard dict to a list instead of adding it to a parent dict.
Step-by-Step Instructions
  1. Copy your lab-4-1.py and rename it to lab-4-2.py.
  2. Update your docstring description to reflect Lab 4.2.
  3. Define load_hazards(filename) that takes a filename parameter.
  4. Create an empty list inside the function: hazards = [].
  5. Open the file and loop through each line.
  6. For each line: use line.strip().split(',') to get a list of parts.
  7. Create a hazard dict: {'name': parts[0], 'heads': parts[1], 'tails': parts[2]}.
  8. Append the dict to the list: hazards.append(hazard).
  9. Close the file and return the hazards list.

Tips & Pitfalls
Common Mistake Fix
Still checking lowercase after .upper() Only compare against 'Y' and 'N', never 'y' or 'n'.
Forgetting the parentheses: .upper Methods require parentheses: .upper().
Reversed order: .upper().strip() Won't crash, but .strip().upper() is the standard pattern. Strip the junk first, then transform.

  • No remaining or patterns for case handling

What You're Building

A function that selects a random hazard from your list, displays it, flips a coin, and shows the appropriate outcome message. Returns True if the player escaped (HEADS) or False if not (TAILS).

Step-by-Step Instructions

  2. Pick a random hazard: hazard = random.choice(hazards).

  4. Flip a coin: result = function_library.flip_coin().

Tips & Pitfalls
Common Mistake Why It Fails
Using hazard['HEADS'] (uppercase key) Keys are lowercase: 'heads' and 'tails', matching how you defined them in load_hazards().
Forgetting to return True or False The caller needs to know if the player escaped. The function must return a boolean.
Using random.choice(hazards) without import random Make sure import random is at the top of your file.
What You Should Have

  • Random hazard selection using random.choice()
  • Correct lowercase dictionary keys: 'heads' and 'tails'

D20 Travel System
Replace instant travel with dice-roll-based progression and random hazard encounters.
What You're Building

A travel_to() function where you roll a D20 each turn, travel roll × 10 miles, and have a 30% chance of encountering a hazard. The journey continues until you've covered the full distance to the destination.

Step-by-Step Instructions
  1. Define travel_to(destination, distance, hazards).
  2. Initialize: remaining = distance, miles_traveled = 0, hazards_faced = 0, hazards_escaped = 0.
  3. Create the travel loop: while remaining > 0:.
  4. Roll the D20: roll = function_library.roll_d20().
  5. Calculate miles this turn: miles = roll * 10.
  6. Cap miles if it would overshoot the destination: if miles > remaining: miles = remaining.
  7. Update accumulators: add miles to miles_traveled, subtract from remaining.
  8. Print progress: show the roll, miles gained this turn, and remaining distance.
  9. Check for a hazard only if remaining > 0: use function_library.check_percentage(30).
  10. If a hazard triggers: call handle_hazard(hazards), increment hazards_faced, and increment hazards_escaped if it returned True.
  11. After the loop, return the tuple: (miles_traveled, hazards_faced, hazards_escaped).
No Hazards on the Last Turn

Only check for hazards when remaining > 0. You don't want a hazard to fire on the turn you've already arrived at your destination.

Step-by-Step Instructions

Update main() to use the new system:

  1. Load hazards alongside city data at the start of main().
  2. Remove the old coin flip destination gate (the D20 travel system replaces it).
  3. Call travel_to() after the player chooses a destination, passing destination name, distance, and hazards list.
  4. Unpack the returned tuple and add to running totals: miles, faced, escaped = travel_to(...).
  5. Update the final report to display total hazards faced and escaped.
Test Your Code

Play through the complete game and verify: D20 rolls determine progress (roll × 10 miles), hazards appear randomly on about 30% of turns, hazard outcomes depend on the coin flip, the final report shows hazards escaped vs. faced, and total miles matches the sum of all journeys.

Tips & Pitfalls
Common Mistake Why It Fails
Using check_percentage(0.3) The function expects a whole number: check_percentage(30).
D20 range wrong: randint(0, 20) A D20 rolls 1–20, not 0–20. Use the library function to be safe.
Forgetting to return the tuple The caller needs (miles, faced, escaped) to update the final report.

  • Returns a tuple of (miles_traveled, hazards_faced, hazards_escaped)
