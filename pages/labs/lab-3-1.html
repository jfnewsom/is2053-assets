Save Your Progress
MODULE 3 • WEEK 1 • BUILD • CHAPTERS 6: FILES AND EXCEPTIONS
Overview

Your road trip game works great — but every time the program ends, all that progress disappears. Maria drove from San Antonio all the way to Houston, racked up 200 miles, survived two hazards, and then the program quit. Gone. Starting over from scratch is not a game. It's a punishment. This week you fix that. You'll use Python's file I/O tools to write your game state to a text file and read it back when the player returns. By the end of this lab, a player can save mid-trip, quit the program, come back tomorrow, and pick up exactly where they left off.

Checkpoint
1
Add the SAVE_FILE Constant and save_game() Function
What You're Building

You are adding the ability to write your game state to a text file. The save_game() function takes the six variables that describe where the player is and what has happened, and writes them to savegame.txt — one value per line.

Step-by-Step Instructions
  1. Start by copying your lab-2-3.py file to a new file named lab-3-1.py. All your work this week goes in lab-3-1.py.
  2. Near the top of your file, after your imports and before the distance constants, add the new constant: SAVE_FILE = 'savegame.txt'
  3. Define a new function save_game() that accepts six parameters: player_name, current_city, total_miles, total_turns, hazards_faced, and hazards_escaped.
  4. Inside save_game(), open the save file for writing: outfile = open(SAVE_FILE, 'w')
  5. Write each variable to the file as its own line. For string variables, write the value plus '\n'. For integer variables, wrap in str() first: outfile.write(str(total_miles) + '\n')
  6. Write all six variables in this exact order: player_name, current_city, total_miles, total_turns, hazards_faced, hazards_escaped.
  7. Close the file with outfile.close() and print 'Game saved!' to confirm.
BookEx Patterns
  • Program 6-1 — file_write.py (Step 1) — Shows the basic open-write-close pattern for creating a text file.
  • Program 6-4 — write_names.py (Step 2) — Demonstrates writing user-provided strings to a file with '\n' appended.
  • Program 6-6 — write_numbers.py (Step 3) — Shows why numbers must be wrapped in str() before calling write().
write() only accepts strings

If you pass an integer directly to write(), Python raises a TypeError. You must convert every number before writing it: outfile.write(str(total_miles) + '\n')

What You Should Have
  1. SAVE_FILE = 'savegame.txt' constant defined near the top of your file
  2. save_game() function defined with six parameters and a complete docstring
  3. All six variables written to the file in order, with integers converted via str()
  4. File closed after writing, and a confirmation message printed
Checkpoint
2
Build the load_game() Function
What You're Building

You are writing the reverse of save_game(). The load_game() function opens savegame.txt, reads the six values back in the same order they were written, and returns them as a tuple. If the file does not exist, it returns None instead of crashing.

Step-by-Step Instructions
  1. Define load_game() with no parameters.
  2. Wrap the entire function body in a try-except FileNotFoundError block. This handles the case where no save file exists yet.
  3. In the try block, open the file for reading: infile = open(SAVE_FILE, 'r')
  4. Read each line in the same order it was written. For string variables, use readline().rstrip('\n'). For integer variables, wrap the readline in int(): total_miles = int(infile.readline())
  5. After reading all six values, close the file with infile.close().
  6. Return all six values as a tuple: return (player_name, current_city, total_miles, total_turns, hazards_faced, hazards_escaped)
  7. In the except FileNotFoundError block, simply return None.
BookEx Patterns
  • Program 6-3 — line_read.py (Step 1) — Shows how readline() reads one line at a time, including the '\n' character.
  • Program 6-5 — strip_newline.py (Step 2) — Demonstrates using rstrip('\n') to clean up lines read from a file.
  • Program 6-7 — read_numbers.py (Step 3) — Shows wrapping readline() in int() to convert file data back to integers.
  • Program 6-28 — display_file2.py (Step 4) — Demonstrates try-except FileNotFoundError to handle a missing file gracefully.
"Order is everything in file I/O. Whatever order you write the lines — that is the exact order you have to read them back. Write name first, read name first. Mix up the order and your city becomes a number and your miles become a name. Test it."

What You're Building

Step-by-Step Instructions

  7. If the player chooses 1, run the existing new-game setup: get their name, set current_city = 'San Antonio', initialize all stats to 0.

Scope Reminder

Permitted Techniques

Chapters in scope: 2–6 (all techniques from Chapters 2–5 plus Chapter 6 file I/O and exceptions as introduced this week)

  • open(filename, 'w') and open(filename, 'r')
  • outfile.write(str(value) + '\n')
  • infile.readline().rstrip('\n')

  • try-except FileNotFoundError
  • Tuple return from a function

Not permitted: with statement, readlines(), list operations, sort(), append(), dictionaries, classes

Do not hardcode the filename 'savegame.txt' inside save_game() or load_game(). Use the SAVE_FILE constant — CodeGrade may rename the file to test flexibility.

Read back every value in the exact same order it was written. If save_game() writes player_name first, load_game() must read player_name first.
