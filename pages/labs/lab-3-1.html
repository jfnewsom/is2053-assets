What You're Building

A function that takes 6 parameters (player_name, current_city, total_miles, total_turns, hazards_faced, hazards_escaped) and writes each one to a file on its own line.

Why This Matters

This is your first time making data persist — it survives after your program ends. The file is just text, so you can open it in Notepad to see exactly what you saved. Human-readable data = easy debugging!

BookEx Patterns
  • Program 6-1 — file_write.py (Step 1) — You wrote outfile.write('John Locke\n') to save a name. Now you'll write player_name the same way — but don't forget + '\n' to put each value on its own line!
Step-by-Step Instructions
  1. Copy your Lab 2.3 code to a new file called lab-3-1.py and update the docstring
  2. Add a constant: SAVE_FILE = 'savegame.txt'
  3. Define save_game() with 6 parameters (see What You're Building)
  4. Open the file for writing: outfile = open(SAVE_FILE, 'w')
  5. Write player_name: outfile.write(player_name + '\n')
  6. Write current_city the same way
  7. Write the 4 numeric values — remember to convert with str()
  8. Close the file: outfile.close()
  9. Print confirmation: 'Game saved!'
Tips & Pitfalls
Mistake Why It Fails
Forgetting '\n' after each write Everything ends up on one line — can't read it back properly
Writing a number directly TypeError! write() only accepts strings. Use str(total_miles)
Forgetting to close the file Data might not save completely — always close()!
Using 'r' instead of 'w' Can't write to a file opened for reading

Build load_game()
Create a function that reads the save file and returns all the game data — or handles a missing file gracefully.
What You're Building

A function that reads 6 lines from the save file, converts them to the right types, and returns them as a tuple. If the file doesn't exist, it returns None.

BookEx Patterns
  • Program 6-5 — strip_newline.py (Step 1) — You used line.rstrip('\n') to remove the trailing newline. Do the same for each readline().
  • Program 6-28 — display_file2.py (Step 2) — You wrapped file operations in try-except FileNotFoundError. Use the same pattern here!

Step-by-Step Instructions
  1. Define load_game() with no parameters (it reads from SAVE_FILE)
  2. Start a try block: try:
  3. Open for reading: infile = open(SAVE_FILE, 'r')
  4. Read player_name: player_name = infile.readline().rstrip('\n')
  5. Read current_city the same way
  6. Read the 4 numbers — wrap in int() to convert
  7. Close the file
  8. Return all 6 values as a tuple: return (player_name, current_city, ...)
  9. Add except block: except FileNotFoundError: return None
Tips & Pitfalls
Mistake Why It Fails
Forgetting rstrip('\n') Strings have invisible '\n' at end — causes weird comparisons
Forgetting int() on numbers Everything from files is a string — math will fail
Reading lines in wrong order You get player_name where you expect total_miles!
No try-except Program crashes if save file doesn't exist

Add the Main Menu
Give players a choice: start fresh, resume their saved game, or quit.
What You're Building

A startup menu that appears before the game begins. If the player chooses "Load," you call load_game() and restore all their variables. If no save exists, fall back to a new game.

Step-by-Step Instructions
  1. In your main() function, add a menu BEFORE any game logic
  2. Display the menu: 1. New Game, 2. Load Saved Game, 3. Quit
  3. Get menu_choice using get_valid_int()
  4. If choice is 3: print goodbye and return to exit
  5. If choice is 2: call load_game() and store the result
  6. If load returns None: print "No save found" and set menu_choice = 1
  7. If load succeeds: unpack the tuple into your 6 variables
  8. If choice is 1: do normal new game setup (get name, initialize variables)
Tips & Pitfalls
Mistake Why It Fails
Unpacking tuple with wrong index saved[0] is player_name, saved[2] is total_miles — order matters!
Checking saved == None Works, but saved is None is the Pythonic way
Not falling back to new game If load fails and you don't handle it, variables are undefined
