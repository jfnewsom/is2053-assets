What You're Building

A function that reads trivia.txt where each city can have multiple trivia questions. Structure: trivia['Houston'] returns a list of question dicts.

Why This Matters

This is the most sophisticated data structure you’ve built yet: a dictionary of lists of dictionaries. Real applications use nested structures like this constantly — think of a playlist app where each artist maps to a list of album objects. Mastering this pattern unlocks serious programming power.

Pro Tip
# The structure you're building:
trivia = {
    'Houston': [
        {'question': 'NASA is in Houston', 'answer': 'TRUE'},
        {'question': 'Houston is the capital', 'answer': 'FALSE'}
    ],
    'Austin': [
        {'question': 'Austin has bats', 'answer': 'TRUE'}
    ]
}
Step-by-Step Instructions
  1. Copy lab-4-2.py and rename it to lab-4-3.py
  2. Update the top-of-file docstring to describe Lab 4.3
  3. Define load_trivia(filename)
  4. Create an empty dict: trivia = {}
  5. Open the file and loop through lines: use line.strip().split(',')
  6. Extract parts: city = parts[0], question = parts[1], answer = parts[2]
  7. Check if the city key exists: if city not in trivia: trivia[city] = []
  8. Append the question dict: trivia[city].append({'question': question, 'answer': answer})
  9. Close the file and return the trivia dict

Trivia Gate
Replace the coin flip with a TRUE/FALSE question system that returns destination, distance, and result.
What You're Building

A function that presents a trivia question, accepts TRUE/FALSE input (using .strip().upper()), and returns the final destination, distance, and whether the player got it correct.

Step-by-Step Instructions
  1. Define trivia_gate(dest, trivia, all_dests, distances, current)
  2. Print a header: --- TRIVIA TIME! --- and announce the destination
  3. Get a question: if dest has trivia, use random.choice(trivia[dest]); otherwise pick from any city in the dict
  4. Display the question and prompt for an answer
  5. Get input: answer = input('TRUE or FALSE? ').strip().upper()
  6. Normalize the answer: accept T/TRUE as 'TRUE', F/FALSE as 'FALSE'; anything else is invalid
  7. If correct: return (dest, distance, True)
  8. If wrong: pick a random other destination and return (redirect, new_distance, False)
Pro Tip

To get “other” destinations for the redirect, loop through all_dests and build a new list that skips the chosen destination. The pattern is the same one you used in Module 3 — loop, check a condition, and .append() if it passes.

What You're Building

"With a list, you’d have to check 'if city not in visited' before adding. With a set? Just .add() it. Duplicates vanish automatically. Less code, fewer bugs — that’s the kind of shortcut I wish I’d known sooner!"

Junior Developer, Bat City Collective

  4. Update handle_hazard() to accept the unique hazards set as a parameter, then add the hazard’s name to it

Tips & Pitfalls
Common Mistake Why It Fails
Creating a set with {} instead of set() {} creates an empty dictionary, not a set
Using .append() instead of .add() Sets use .add(), lists use .append() — don’t mix them up
Forgetting to pass unique_hazards to handle_hazard() The set won’t track hazards if the function can’t access it
