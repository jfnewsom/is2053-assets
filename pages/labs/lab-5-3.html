Why This Matters

Inheritance is how professional codebases stay maintainable. Instead of duplicating code, you build hierarchies where specialized classes extend base classes. Every game engine, web framework, and mobile app uses this pattern. Master it here and you're ready for software engineering.

"Without inheritance, you'd have to copy the entire City class, paste it, rename it WinCity, then add your beach stuff. Every time you fix a bug in City, you'd fix it again in WinCity. Nightmare. Inheritance means WinCity automatically gets City's code. Fix City once, WinCity gets the fix for free."
Bobby Hernandez
Director of Technology, Bat City Collective

BookEx Patterns
  • Program 11-9 — animals.py (Step 1) — You typed class Dog(Mammal): to make Dog inherit from Mammal, then called Mammal.__init__(self, species) inside Dog's __init__. WinCity follows the exact same pattern — inherit from City, call City's constructor.
Step-by-Step Instructions
  1. Copy your city.py from Lab 5.2 (keep the entire City class exactly as-is)
  2. At the BOTTOM of city.py (after the City class ends), add the WinCity class:
class WinCity(City):
    """
    A winning destination city (beach!) that inherits from City.

    Adds beach-specific features while keeping all City functionality.
    """

    def __init__(self, name, tagline, beach_activities):
        """
        Initialize a WinCity.

        Args:
            name (str): City name
            tagline (str): City motto
            beach_activities (list): List of things to do at the beach
        """
        # Call the PARENT class constructor
        City.__init__(self, name, 'WIN', tagline)

        # Add WinCity-specific attribute
        self.__beach_activities = beach_activities
  1. Notice: we pass 'WIN' as the city_type to City — WinCity is ALWAYS a WIN type, so the student doesn't have to specify it!

BookEx Patterns
  • Program 11-9 — animals.py (Step 1) — Remember how Dog had make_sound() that returned 'Woof!' while Cat returned 'Meow!'? Same method name, different behavior. That's method overriding. WinCity will override get_arrival_message() to return a beach celebration!
Step-by-Step Instructions
  1. First, add get_arrival_message() to the City class (base version — regular cities need arrival messages too!):
# In the City class:
def get_arrival_message(self):
    """Return arrival message for this city."""
    return f'Welcome to {self.get_name()}!'
  1. Add get_beach_activities() to WinCity:
def get_beach_activities(self):
    """Return list of beach activities."""
    return self.__beach_activities.copy()
  1. Add display_beach_activities() to WinCity:
def display_beach_activities(self):
    """Display available beach activities."""
    print('Things to do at the beach:')
    for activity in self.__beach_activities:
        print(f'  - {activity}')
  1. Override get_arrival_message() in WinCity — same method name, beach-specific behavior:
def get_arrival_message(self):
    """Override: Special beach celebration message!"""
    return f'YOU MADE IT TO {self.get_name().upper()}! THE BEACH AWAITS!'

Tips & Pitfalls
Common Mistake Why It Fails
Returning the actual __beach_activities list Return a copy with list(self.__beach_activities) to protect the original
Overriding method with wrong signature Override must have same parameters: get_arrival_message(self)
Adding get_arrival_message() only to WinCity Base City class needs it too — regular cities also need arrival messages
