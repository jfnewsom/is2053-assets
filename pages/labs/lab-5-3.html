Create the WinCity Subclass

Add a WinCity class to city.py that inherits from City.

A WinCity class that is a City but with an extra attribute for beach activities. One line—class WinCity(City):—gives you all of City's powers instantly.

Inheritance is how professional codebases stay maintainable. Instead of duplicating code, you build hierarchies where specialized classes extend base classes. Every game engine, web framework, and mobile app uses this pattern. Master it here and you're ready for software engineering.

"Without inheritance, you'd have to copy the entire City class, paste it, rename it WinCity, then add your beach stuff. Every time you fix a bug in City, you'd fix it again in WinCity. Nightmare. Inheritance means WinCity automatically gets City's code. Fix City once, WinCity gets the fix for free."

Director of Technology, Bat City Collective

BookEx 11.1.1 (animals.py): You typed class Dog(Mammal): to make Dog inherit from Mammal. Then you called Mammal.__init__(self, species) inside Dog's __init__. WinCity follows the exact same pattern—inherit from City, call City's constructor.

  1. Copy your city.py from Lab 5.2 (or use your working 5.1 version)
  2. At the BOTTOM of city.py (after the City class), add the WinCity class: 
    class WinCity(City):
    """
    A winning destination city (beach!) that inherits from City.
    
    Adds beach-specific features while keeping all City functionality.
    """
    
    def __init__(self, name, tagline, beach_activities):
        """
        Initialize a WinCity.
        
        Args:
            name (str): City name
            tagline (str): City motto
            beach_activities (list): List of things to do at the beach
        """
        # Call the PARENT class constructor
        City.__init__(self, name, 'WIN', tagline)
        
        # Add WinCity-specific attribute
        self.__beach_activities = beach_activities
  3. Notice: We pass 'WIN' as the city_type to City—WinCity is ALWAYS a WIN type!
Mistake Why It Fails
class WinCity: No inheritance! WinCity won't have City's methods
City.__init__(name, ...) Forgot self! Must pass self as first argument
City.__init__(self, name, ...) Correct! Passes self to parent constructor

Beach Methods & Method Override

Add beach-specific methods and override get_arrival_message().

A getter for beach activities and an overridden get_arrival_message() that returns a special beach celebration message instead of the generic City message.

BookEx 11.1.1 (animals.py): Remember how Dog had make_sound() that returned 'Woof!' while Cat returned 'Meow!'? Same method name, different behavior. That's method overriding. WinCity will override get_arrival_message() to return a beach celebration!

  1. First, add get_arrival_message() to your City class (if not already there): 
    # In the City class:
def get_arrival_message(self):
    """Return arrival message for this city."""
    return f'Welcome to {self.get_name()}!'
  2. Add get_beach_activities() to WinCity: 
    def get_beach_activities(self):
    """Return list of beach activities."""
    return self.__beach_activities.copy()
  3. Add display_beach_activities() to WinCity: 
    def display_beach_activities(self):
    """Display available beach activities."""
    print('Things to do at the beach:')
    for activity in self.__beach_activities:
        print(f'  - {activity}')
  4. Override get_arrival_message() in WinCity: 
    def get_arrival_message(self):
    """Override: Special beach celebration message!"""
    return f'YOU MADE IT TO {self.get_name().upper()}! THE BEACH AWAITS!'
