Nested Houston Logic
What You're Building

The heart of this assignment: if the player reaches Houston, they can choose to continue to Galveston. This is an if inside an if — a nested if.

BookEx Patterns
  • Program 3-5 — loan_qualifier.py — Shows nested if — the inner if only runs when the outer if is true. Study how the indentation levels work. That's exactly what you need here.
  • Program 3-7 — loan_qualifier2.py — Shows the or operator in conditions. The pattern you need: choice == 'y' or choice == 'Y' — you must repeat the full comparison on both sides.
"When I first learned the or operator, I kept writing 'if choice == "y" or "Y":' and wondering why it always ran. You need the full comparison on both sides: 'choice == "y" or choice == "Y"'. Trust me, this trips everyone up at first!"
Tyler Nguyen
Junior Developer, Bat City Collective
Step-by-Step Instructions

Replace your Houston branch (elif choice == 2) with nested logic. The structure is:

  1. Check if made it to Houston (miles_remaining >= SAT_TO_HOUSTON)
  2. If yes: subtract SAT_TO_HOUSTON from miles_remaining, set total_distance, then ask 'Continue to Galveston? (y/n): '
  3. If they say y/Y: check if miles_remaining >= HOU_TO_GALVESTON — set status WINNER or SHORT accordingly
  4. If they say anything else: set status to 'STOPPED'
  5. If didn't make Houston: set status to 'SHORT', set total_distance to SAT_TO_HOUSTON
Tips & Pitfalls
Mistake Why It Fails
choice == 'y' or 'Y' Always True! Need: choice == 'y' or choice == 'Y'
Wrong indentation level Nested code must be indented further than the outer if
Not updating miles_remaining Must subtract SAT_TO_HOUSTON before checking if you can reach Galveston
