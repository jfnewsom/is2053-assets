Nested Houston Logic

The heart of this assignment: if you reach Houston, ask about continuing to Galveston.

BookEx 3.4.1 (loan_qualifier.py): Shows nested if—an if inside another if. The inner if only runs when the outer if is true.

"When I first learned the or operator, I kept writing if choice == 'y' or 'Y': and wondering why it always ran. You need the full comparison on both sides: choice == 'y' or choice == 'Y'. Trust me, this trips everyone up at first!"

Junior Developer, Bat City Collective

Nested decision logic: if the player reaches Houston, they can choose to continue to Galveston. This is an if inside an if—a nested if.

Replace your Houston branch (elif choice == 2) with nested logic. The structure is:

  • Check if made it to Houston (miles_remaining >= 0)
  • If yes: ask "Continue to Galveston? (y/n)"
  • If they say y/Y: try for Galveston, check if made it
  • If they say anything else: stay in Houston (STOPPED)
  • If didn't make Houston: show miles short (SHORT)

Test all 4 Houston outcomes:

  • Roll 5, choice 2: Didn't make Houston → SHORT
  • Roll 20, choice 2, 'n': Made Houston, stayed → STOPPED
  • Roll 20, choice 2, 'y': Made Houston, reached Galveston → WINNER
  • Roll 15, choice 2, 'y': Made Houston, short of Galveston → SHORT
Mistake Why It Fails
choice == 'y' or 'Y' Always True! Need: choice == 'y' or choice == 'Y'
Wrong indentation level Nested code must be indented further than outer if
Not updating miles_remaining Need to subtract distance traveled before checking Galveston
  • Nested if inside the Houston branch
  • Input prompt asking about Galveston
  • Logic handling y/Y for continue, anything else for stop
  • All 4 Houston outcomes working correctly
