Build the CITY_DISTANCES Dictionary
What You're Building

You're replacing 20+ individual distance constants with a single nested dictionary. This is the core data structure upgrade for Module 4.

Step-by-Step Instructions
  1. Create a constant CITY_DISTANCES near the top of your file.
  2. Add San Antonio as the first key: CITY_DISTANCES = {'San Antonio': {...}}
  3. Add all five destination cities as nested keys with integer mile values.
Textbook Reference
  • Program 9-1 — dictionary1.py (Step 1) — Shows basic dictionary creation with string keys and integer values.
  • Program 9-2 — dictionary2.py (Step 2) — Demonstrates key access and iteration. 
CITY_DISTANCES = {
    'San Antonio': {
        'Austin':   80,
        'Houston':  197,
        'Dallas':   275
    }
}
Expected Output
Expected Output
Where would you like to travel?
  A) Austin     (80 miles)
  B) Houston    (197 miles)
  C) Dallas     (275 miles)
Why This Matters

One lookup replaces ten branches. This is the moment your code stops being a long if/elif chain and starts being data-driven.

Tips and Pitfalls
Tips & Pitfalls
Mistake Why It Fails
Using CITY_DISTANCES['austin'] Dictionary keys are case-sensitive — use exact capitalization
Using an if/elif chain instead of dict access Defeats the purpose of a dictionary; CodeGrade will flag it
Key Case Matters

Dictionary keys are case-sensitive. 'austin' and 'Austin' are different keys.

You're Done When...

Your program looks up distances using CITY_DISTANCES[city][destination] with no hardcoded numbers in the game loop.

Test in Isolation

Before wiring the dictionary into the game, test it by itself.

  • Open a new file and create just the dictionary
  • Print a few lookups to confirm the values are right
  • Then copy it into your main file
Why Nested Dicts?

A nested dictionary models a two-dimensional lookup: from city → to destination → miles. It's the same concept as a table with row and column headers.

Scope Violation

Do not use list comprehensions or try/except — those are Module 4.2 and 4.3 topics. CodeGrade's AST check will flag them.
