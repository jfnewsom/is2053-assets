Overview

In Module 1, you chose a destination from San Antonio — but never actually went anywhere! Time to hit the road. This week, you'll add a while loop that simulates driving toward your destination. Initialize Tracking Variables
What You're Building

Three variables: one to track how far you still need to go, one to accumulate total miles traveled, and one to count turns.

Why This Matters

The #1 accumulator mistake is initializing inside the loop. If you set total = 0 inside the loop, it resets to 0 every iteration! You must initialize BEFORE the loop.

BookEx Patterns
  • Program 4-17 — sum_numbers.py — You initialized total = 0.0 BEFORE the loop, then added to it inside. Same pattern here — initialize your tracking variables before the loop!
Step-by-Step Instructions

After your if-elif-else chain (where destination and distance are set), add three variables:

  1. distance_remaining — starts equal to the full distance
  2. total_miles — accumulator, starts at 0
  3. turn — counter, starts at 1

Tips & Pitfalls
Mistake Why It Fails
Initializing variables INSIDE the loop They reset every iteration! Initialize BEFORE the loop starts
Starting turn at 0 instead of 1 Turn 1 is the first turn — human counting starts at 1!
Putting tracking variables before if-elif-else You need to know distance first before setting distance_remaining
What You Should Have

  2. distance_remaining equals the distance for the chosen destination
  3. total_miles is 0, turn is 1

Build the while Loop
What You're Building

A while loop that continues as long as there's still distance to travel. Think of it as: "Keep driving WHILE there are miles remaining."

BookEx Patterns
  • Program 4-2 — temperature.py — You wrote a loop that continued while temperature > MAX_TEMP. Your travel loop continues while distance_remaining > 0. Same pattern!
Step-by-Step Instructions
  1. Add a "journey begins" header with print statements before the loop
  2. Write the while statement: continue while distance_remaining is greater than 0
  3. Don't forget the colon — while condition: requires it
  4. For now, just add a placeholder inside the loop: print the turn number
Infinite Loop Danger!

If you run this now without updating distance_remaining inside the loop, it will run forever. Press Ctrl+C to stop it. Use the test harness below to safely test your loop structure.

Mistake Why It Fails
Using while distance_remaining != 0 You might overshoot 0! Use > 0 instead

Complete the Loop Body
What You're Building

The complete loop body that: shows turn info, travels 50 miles, updates distance_remaining so the loop eventually ends, accumulates total_miles, and increments the turn counter.

Step-by-Step Instructions

Inside your while loop (indented!), do these things in order:

  1. Display the turn header — show what turn number this is
  2. Display miles remaining — so the player sees their progress
  3. Display a travel message — "You travel 50 miles..." (use your constant!)
  4. Update distance_remaining — SUBTRACT miles traveled
  5. Update total_miles — ADD miles traveled (accumulator pattern!)
  6. Increment turn — add 1 so next iteration shows Turn 2, Turn 3, etc.
BookEx Patterns
  • Program 4-17 — sum_numbers.py — You wrote total = total + number to accumulate. Your version: total_miles = total_miles + MILES_PER_TURN
Tips & Pitfalls
Mistake Why It Fails
Forgetting to subtract from distance_remaining Infinite loop! Distance never reaches 0
Using = instead of = ... + for accumulator Overwrites instead of accumulating — always shows 50
Wrong indentation Code outside loop only runs once; code inside repeats
Forgetting to increment turn Every turn displays as Turn 1

Arrival & Trip Summary
What You're Building

Code that runs after the while loop (fully unindented!) to announce arrival and display trip statistics.

Why This Matters

Code INSIDE the loop repeats. Code AFTER the loop (unindented) runs once when the loop is done. This is where you celebrate arrival and show the summary.

"The Trip Summary is your user's reward for completing the journey! Make it feel like an achievement — use separator lines, clear labels, and organized data. A well-designed summary turns raw numbers into a story of their adventure."
Sasha Mireles
Creative Director, Bat City Collective
Step-by-Step Instructions

After the while loop (no indentation!), add:

  1. Arrival announcement — "You made it to {destination}!"
  2. Trip Summary header with separator lines
  3. Display Driver (player_name), Destination, Distance, Total turns, and Total miles
  4. Display total turns as turn - 1 (see warning below)
  5. Outcome message — different text for Corpus Christi (beach!) vs other destinations
Why turn - 1?

You increment turn at the END of each loop iteration. After 3 turns complete, turn equals 4 when the loop exits. So display turn - 1 for the actual count.
