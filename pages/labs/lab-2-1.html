Initialize Tracking Variables

Set up the variables you'll need to track your journey BEFORE the loop starts.

Three variables: one to track how far you still need to go, one to accumulate total miles traveled, and one to count turns.

The #1 accumulator mistake is initializing inside the loop. If you set total = 0 inside the loop, it resets to 0 every iteration! You must initialize BEFORE the loop.

BookEx 4.4 (sum_numbers.py), step 2: You initialized total = 0.0 BEFORE the loop, then added to it inside. Same pattern here—initialize your tracking variables before the loop!

Common Mistake Why It Fails
Initializing variables INSIDE the loop They reset every iteration! Initialize BEFORE the loop starts
Starting turn at 0 instead of 1 Turn 1 is the first turn—human counting starts at 1!
Putting tracking variables before if-elif-else You need to know the distance FIRST before setting distance_remaining
  • Three tracking variables initialized AFTER if-elif-else, BEFORE the loop
  • distance_remaining equals the distance for the chosen destination
  • total_miles is 0, turn is 1

Common Mistake Why It Fails
Using while distance_remaining != 0 You might overshoot 0! Use > 0 instead
Forgetting the colon after the while condition SyntaxError—while condition: needs that colon!
Loop body not indented Everything INSIDE the loop must be indented one level

Inside your while loop (indented!), do these things in order:

Why turn - 1? You increment turn at the END of each loop iteration. After 3 turns complete, turn equals 4 when the loop exits. So display turn - 1 for the actual count.
