Build get_destination() and get_redirect()

Replace your if/elif choice handlers with elegant list indexing.

Two functions: get_destination(city, choice) returns the destination name and distance using [choice - 1] indexing. get_redirect(city, avoid) picks a random different destination for detours.

For get_destination(city, choice):

  1. Define function with two parameters: city and choice
  2. Calculate index: index = choice - 1
  3. Use if/elif to determine which city's lists to access
  4. Return a tuple: return SA_DESTINATIONS[index], SA_DISTANCES[index]
  5. Add else case returning 'San Antonio', 100

For get_redirect(city, avoid):

  1. Define function with two parameters: city and avoid
  2. Use if/elif to select city lists, then make copies: destinations = SA_DESTINATIONS.copy()
  3. Check if avoid is in the list: if avoid in destinations:
  4. Find its position: idx = destinations.index(avoid)
  5. Remove from both copies: destinations.remove(avoid) and distances.pop(idx)
  6. Pick random from remaining using random.randint(0, len(destinations) - 1)

BookEx 7.4: You used the in operator to check if a product number existed in a list. Same pattern here: if avoid in destinations:

BookEx 7.5.2 & 7.5.3: You used index() to find an item's position and remove() to delete it. For parallel lists, you also need pop(idx) to remove by position.

CRITICAL: Why .copy() is Essential

If you write destinations = SA_DESTINATIONS, both variables point to the same list. When you remove an item, you delete it from your original constant! Always use .copy() when you need to modify a list without affecting the original.

Add Tracking Lists & Enhanced Save/Load

Track your route and hazards in lists, then update save/load to preserve them.

Two tracking lists (visited_cities and hazards_list) that grow as you play. Updated save_game() and load_game() that preserve these lists using a ---HAZARDS--- separator.

Part A: Initialize Tracking Lists

  1. In new game setup: visited_cities = ['San Antonio'] (start with home city)
  2. Also add: hazards_list = [] (empty until hazards happen)
  3. After arriving at destination: visited_cities.append(destination)
  4. When hazard occurs: hazards_list.append(hazard) (where hazard is the message string)

Part B: Update save_game()

  1. Add two new parameters: visited_cities and hazards_list
  2. After writing basic stats, loop through visited_cities and write each city
  3. Write the separator: outfile.write('---HAZARDS---\n')
  4. Loop through hazards_list and write each hazard

Part C: Update load_game()

  1. After reading basic stats, create empty visited_cities = []
  2. Use a while loop to read lines until you hit '---HAZARDS---'
  3. Append each city to visited_cities
  4. Create empty hazards_list = []
  5. Continue reading and appending until end of file
