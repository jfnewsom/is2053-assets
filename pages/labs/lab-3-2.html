Build get_destination() and get_redirect()
Replace your if/elif choice handlers with elegant list indexing.
What You're Building

Two functions: get_destination(city, choice) returns the destination name and distance using [choice - 1] indexing. get_redirect(city, avoid) picks a random different destination for detours.

Step-by-Step Instructions
  1. For get_destination(city, choice): Define function with two parameters: city and choice
  2. Calculate index: index = choice - 1
  3. Use if/elif to determine which city's lists to access
  4. Return a tuple: return SA_DESTINATIONS[index], SA_DISTANCES[index]
  5. Add else case returning 'San Antonio', 100
  6. For get_redirect(city, avoid): Define function with two parameters: city and avoid
  7. Use if/elif to select city lists, then make copies: destinations = SA_DESTINATIONS.copy()
  8. Check if avoid is in the list: if avoid in destinations:
  9. Find its position: idx = destinations.index(avoid)
  10. Remove from both copies: destinations.remove(avoid) and distances.pop(idx)
  11. Pick random from remaining using random.randint(0, len(destinations) - 1)
BookEx Patterns
  • Program 7-2 — in_list.py (Step 1) — You used the in operator to check if a product number existed in a list. Same pattern here: if avoid in destinations:
  • Program 7-4 — index_list.py (Step 2) — You used index() to find an item's position and remove() to delete it. For parallel lists, you also need pop(idx) to remove by position.
CRITICAL: Why .copy() is Essential

If you write destinations = SA_DESTINATIONS, both variables point to the same list. When you remove an item, you delete it from your original constant! Always use .copy() when you need to modify a list without affecting the original.

Add Tracking Lists & Enhanced Save/Load
Track your route and hazards in lists, then update save/load to preserve them.
What You're Building

Two tracking lists (visited_cities and hazards_list) that grow as you play. Updated save_game() and load_game() that preserve these lists using a ---HAZARDS--- separator.

Step-by-Step Instructions
  1. Part A — Initialize Tracking Lists. In new game setup: visited_cities = ['San Antonio'] (start with home city). Also add: hazards_list = [] (empty until hazards happen). After arriving at destination: visited_cities.append(destination). When hazard occurs: hazards_list.append(hazard) (where hazard is the message string).
  2. Part B — Update save_game(). Add two new parameters: visited_cities and hazards_list. After writing basic stats, loop through visited_cities and write each city. Write the separator: outfile.write('---HAZARDS---\n'). Loop through hazards_list and write each hazard.
  3. Part C — Update load_game(). After reading basic stats, create empty visited_cities = []. Use a while loop to read lines until you hit '---HAZARDS---'. Append each city to visited_cities. Create empty hazards_list = []. Continue reading and appending until end of file. Return tuple now includes both lists.
  4. Part D — Add File Functions to function_library.py. These three utility functions live in function_library.py, not in your main game file. Add them under a new # FILE FUNCTIONS comment block after your existing validation functions.
BookEx Patterns
  • Program 7-3 — list_append.py (Step 1) — You built a list dynamically with name_list = [] then name_list.append(name). Same pattern here — start empty, append as events happen!
  • Program 7-16 — write_number_list.py (Step 2) — Writing a list to a file one item per line — exactly what save_list_to_file() does.
  • Program 7-17 — read_number_list.py (Step 3) — Reading a file back into a list — that's load_list_from_file(). The readline() while loop pattern is identical to your load_game() from Lab 3.1.

For function_library.py — save_list_to_file(filename, data_list): Define the function with two parameters: filename and data_list. Open the file for writing: with open(filename, 'w') as outfile:. Loop through data_list and write each item: outfile.write(str(item) + '\n'). No return value needed.

For function_library.py — load_list_from_file(filename): Define the function with one parameter: filename. Create an empty result list: result = []. Open the file inside a try block: with open(filename, 'r') as infile:. Use a readline() while loop — same pattern as your load_game(). Append each line stripped of \n to result. In the except FileNotFoundError block: leave result as empty list. Return result.

For function_library.py — file_exists(filename): Define the function with one parameter: filename. In a try block: attempt to open the file for reading. If it opens successfully: return True. In the except FileNotFoundError block: return False.

Why the ---HAZARDS--- marker?

The ---HAZARDS--- marker separates the variable-length visited_cities list from the variable-length hazards_list. Since both lists can be any length, you need a clear signal for where one ends and the other begins.
