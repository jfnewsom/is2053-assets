Expand the Library

Add your first input validation function to function_library.py.

A new function called get_valid_int() that takes a prompt and a valid range, then keeps asking until the user enters a valid number. This is the input validation loop pattern from Chapter 4.6!

Right now, if someone enters "6" or "banana" at your menu, your game crashes. Real software never trusts user input. This function catches bad input and keeps asking until the user gets it right.

  1. Open your function_library.py from Lab 2.2
  2. Add a comment: # VALIDATION FUNCTIONS (Added in Lab 2.3)
  3. Define get_valid_int(prompt, min_val, max_val) with three parameters
  4. Priming read: value = int(input(prompt))
  5. Validation loop: while value < min_val or value > max_val:
  6. Inside loop: Print error, then re-read with value = int(input(prompt))
  7. After loop: return value
  8. Add a docstring explaining parameters and return value

BookEx 4.6 (retail_with_validation.py): You practiced the input validation loop—priming read, while loop checks condition, re-read inside. Same pattern here, wrapped in a reusable function! BookEx 5.5: Shows how to pass values INTO a function using parameters.

Mistake Why It Fails
Forgetting re-read inside loop Infinite loop! Value never changes
Using and instead of or Logic error: can't be BOTH < min AND > max
Returning inside the loop Returns before validation—defeats purpose!
Missing priming read before while NameError: 'value' not defined

Add Yes/No Validation

Add a function to validate yes/no responses and return a boolean.

A function called get_yes_no() that accepts 'y', 'Y', 'n', or 'N' and returns True for yes or False for no.

  1. Define get_yes_no(prompt) with one parameter
  2. Priming read: answer = input(prompt)
  3. Validation loop: Continue while answer is NOT 'y', 'Y', 'n', or 'N'
  4. Inside loop: Print error message, then re-read
  5. After loop: Return True if answer is 'y' or 'Y', otherwise return False
  6. Add a docstring

BookEx 4.6: Same validation pattern, but checking for specific characters. BookEx 5.8: Shows how to return a value—here you return a boolean (True/False).

Mistake Why It Fails
Using and to check valid options Always rejects: can't equal 'y' AND 'n' AND 'Y' AND 'N'
Forgetting uppercase 'Y' and 'N' Rejects valid input—users expect case-insensitivity
Returning 'y' instead of True Function should return boolean for while conditions
Checking only lowercase in return Returns False for 'Y' even though it means yes
