Expand the Library
What You're Building

A new function called get_valid_int() that takes a prompt and a valid range, then keeps asking until the user enters a valid number. This is the input validation loop pattern from Chapter 4.6, wrapped in a reusable function!

Why This Matters

Right now, if someone enters "6" or "banana" at your menu, your game crashes. Real software never trusts user input. This function catches bad input and keeps asking until the user gets it right.

Step-by-Step Instructions
  1. Open your function_library.py from Lab 2.2
  2. Add a comment: # VALIDATION FUNCTIONS (Added in Lab 2.3)
  3. Define get_valid_int(prompt, min_val, max_val) with three parameters and a docstring explaining each
  4. Priming read: value = int(input(prompt))
  5. Validation loop: while value < min_val or value > max_val:
  6. Inside the loop: print an error message, then re-read with value = int(input(prompt))
  7. After the loop: return value
BookEx Patterns
  • Program 4-21 — retail_with_validation.py — You practiced the input validation loop — priming read, while loop checks condition, re-read inside. Same pattern here, wrapped in a reusable function!
  • Program 5-7 — cups_to_ounces.py — Shows how to pass values into a function using parameters. Your get_valid_int() uses three parameters the same way.

Play Again Loop
"Before you start this checkpoint, take a breath and look at the big picture. You're about to add a SECOND loop to your program. The travel loop from 2.2 — the one with D20 rolls and hazards — isn't going away. It's going INSIDE a bigger loop. Think of it like this: the outer loop controls the whole road trip. The inner loop controls one leg of the trip. Once you see how they nest, the rest clicks into place."
Danny Cavanaugh
Senior Developer, Bat City Collective
What You're Building

A sentinel-controlled loop that wraps your entire game, asking "Play again?" after each trip. Your program now has two loops that work together:

Section What Happens How Often
Welcome + Setup Print title, get player name, initialize totals, set keep_playing = True Once (before any loop)
Show menu + get choice Display destination options, use get_valid_int() for selection Once per leg (outer loop)
Destination Gate Coin flip — did Texas cooperate? Once per leg (outer loop)
Travel loop (D20 + hazards) Roll D20, move miles, check for hazards — same loop from Lab 2.2 Every turn within each leg (inner loop)
Check outcome + play again Trip summary, then ask play again using get_yes_no() Once per leg (outer loop)
Session report Print accumulated stats from ALL legs Once (after game loop ends)
Key insight

The travel loop from Lab 2.2 is unchanged — it just moves inside the game loop. Dice rolls and hazards happen every turn, on every leg. You are not replacing anything — you are wrapping it.

Step-by-Step Instructions
  1. Before the menu, initialize: keep_playing = True
  2. Add while keep_playing: to wrap the game logic
  3. Indent all game code inside the while loop
  4. After the trip summary (inside the loop): keep_playing = function_library.get_yes_no('Play again? (y/n): ')
  5. "Thanks for playing!" message goes outside the loop
BookEx Patterns
  • Program 4-1 — commission.py — You used a sentinel-controlled loop with keep_going = 'y'. Same pattern here, but using your get_yes_no() function to get a clean boolean sentinel!
Tips & Pitfalls
Mistake Why It Fails
Wrong indentation on game code Code runs once then exits, or doesn't run at all
Asking 'play again?' after every turn Prompt is inside the travel loop instead of after the trip summary
'Thanks for playing' inside the loop Message appears after every game, not just at the end
Using while keep_playing == True: Works but redundant — just use while keep_playing:

Step-by-Step Instructions

  2. Inside the loop, after each trip summary, update the accumulators: add 1 to games_played, add total_miles to session_miles, add total_hazards to session_hazards
