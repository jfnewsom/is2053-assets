Connection Methods

Add methods to manage city connections—adding routes and checking distances.

Three methods that let cities know about their road connections. This replaces the old CITY_DISTANCES dictionary—now each city manages its own connections!

  1. Add a comment # --- Connection Methods ---
  2. Define add_connection(self, city_name, distance) that stores the connection: 
    def add_connection(self, city_name, distance):
    """Add a connection to another city."""
    self.__connections[city_name] = distance
  3. Define get_distance_to(self, city_name) that returns the distance to a city, or -1 if not connected: 
    def get_distance_to(self, city_name):
    """Get distance to a connected city. Returns -1 if not connected."""
    return self.__connections.get(city_name, -1)
  4. Define get_destination_names(self) that returns a list of connected city names: 
    def get_destination_names(self):
    """Return list of connected city names."""
    return list(self.__connections.keys())

Step 1: Add this test code:

# DEBUG - DELETE BEFORE CHECKPOINT 4

san_antonio.add_connection('Houston', 200)

print('Testing connection methods:')
print(f'  Distance to Houston: {san_antonio.get_distance_to("Houston")} miles')
print(f'  Distance to Dallas: {san_antonio.get_distance_to("Dallas")} miles')
print(f'  Destinations: {san_antonio.get_destination_names()}')

Common Mistake Why It Fails
Using self.__connections[city_name] for get_distance_to KeyError if city not connected—use .get(city_name, -1) for safe access
Returning self.__connections.keys() directly Returns a dict_keys object, not a list—wrap with list()
Forgetting to add docstrings to methods CodeGrade checks for documentation—every method needs a docstring
  • add_connection() method
  • get_distance_to() returning distance or -1
  • get_destination_names() returning a list

Status Methods & __str__

Add methods to check city type and a __str__ method for nice printing.

Methods that let you ask a city about itself: "Are you a win city?" "Are you a dead end?" Plus a display method and the special __str__ method that Python calls when you print() a City object.

Program 10-9 (bankaccount2.py): You typed a __str__ method that returned a formatted string. Python automatically calls this when you print() an object. Your City's __str__ should return something like 'San Antonio (START)'.

  1. Add a comment # --- Status Methods ---
  2. Define is_win_city(self) that returns True if self.__city_type == 'WIN'
  3. Define is_dead_end(self) that returns True if self.__city_type == 'DEAD_END'
  4. Add a comment # --- Display Methods ---
  5. Define display_destinations(self) that prints the menu of destinations: 
    def display_destinations(self):
    """Display available destinations from this city."""
    print(f'From {self.__name}, you can travel to:')
    option_num = 1
    for dest_name, distance in self.__connections.items():
        print(f'  {option_num}. {dest_name} - {distance} miles')
        option_num = option_num + 1
  6. Define __str__(self) that returns a string like 'San Antonio (START)': 
    def __str__(self):
    """Return string representation of the city."""
    return f'{self.__name} ({self.__city_type})'
