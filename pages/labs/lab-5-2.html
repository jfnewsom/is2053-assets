# ---- player.py ----
class Player:

    # --- Constructor ---   [CP1]
    __init__(self, name, starting_city)
        self.__name              = name
        self.__current_city      = starting_city
        self.__total_miles       = 0
        self.__visited_cities    = [starting_city]   (list -- preserves order)
        self.__unique_cities     = set()             (set -- no duplicates)
        self.__unique_cities.add(starting_city)
        self.__hazards_faced     = 0
        self.__hazards_escaped   = 0
        self.__unique_hazards    = set()
        self.__trivia_asked      = 0
        self.__trivia_correct    = 0

    # --- Getter Methods ---   [CP2]
    get_name()                  returns self.__name
    get_current_city()          returns self.__current_city
    get_total_miles()           returns self.__total_miles
    get_visited_cities()        returns self.__visited_cities.copy()
    get_unique_cities()         returns self.__unique_cities.copy()
    get_unique_city_count()     returns len(self.__unique_cities)
    get_hazards_faced()         returns self.__hazards_faced
    get_hazards_escaped()       returns self.__hazards_escaped
    get_unique_hazards()        returns self.__unique_hazards.copy()
    get_unique_hazard_count()   returns len(self.__unique_hazards)
    get_trivia_asked()          returns self.__trivia_asked
    get_trivia_correct()        returns self.__trivia_correct

    # --- Travel Methods ---   [CP3]
    travel_to(city_name, distance)
        updates: __current_city, __total_miles, __visited_cities, __unique_cities

    # --- Hazard Methods ---   [CP3]
    record_hazard(hazard_name, escaped)
        updates: __hazards_faced, __unique_hazards
        if escaped: updates __hazards_escaped

    # --- Trivia Methods ---   [CP3]
    record_trivia(correct)
        updates: __trivia_asked
        if correct: updates __trivia_correct

    # --- Display Methods ---   [CP4]
    get_route_string()          ' -> '.join(self.__visited_cities)
    get_trivia_accuracy()       returns percentage string e.g. '75.0%'
    __str__                     returns 'Maria in Houston (450 miles)'

Getter Methods
Add all twelve getter-style methods.
What You're Building

Twelve getter methods that provide read access to the Player's private data. These are the methods lab-5-2.py calls to build the final report. Same pattern you used in City -- each getter just returns its attribute.

Why This Matters

Getters enforce a contract: outside code never touches private data directly. lab-5-2.py doesn't reach into player.__total_miles — it calls player.get_total_miles(). That one layer of separation is what makes it safe to change how the Player stores data internally without breaking anything that uses it. Two of the getters — get_unique_city_count() and get_unique_hazard_count() — return len() instead of the raw attribute. The caller gets a number, not a set. That is encapsulation doing real work.

What You're Building

Three methods that replace the scattered update statements in lab-5-1.py's main() function. Each method handles multiple related updates in one place. Instead of four lines of code to record travel, it's one method call: player.travel_to(dest, distance).

Why This Matters

In lab-5-1.py, recording a hazard meant four separate lines in main(): increment the counter, add to the set, check a boolean, maybe increment escaped. Now it is one call: player.record_hazard(name, escaped). The Player knows how to update itself. The caller just says what happened. This is what object-oriented design means in practice — behavior lives with the data it operates on.

"Three methods. travel_to(), record_hazard(), record_trivia(). Together they replace about twenty lines of scattered updates in main(). And next time we want to add a stat — say, a fuel tracker — we add one attribute and one method. We don't go hunting through main() trying to remember every place it touches."

def travel_to(self, city_name, distance):
    """
    Travel to a new city.

    Replaces in main():
        current_city = final_dest
        total_miles = total_miles + miles
        visited.append(final_dest)
        unique_cities.add(final_dest)
    """
    self.__current_city = city_name
    self.__total_miles = self.__total_miles + distance
    self.__visited_cities.append(city_name)
    self.__unique_cities.add(city_name)

  4. Define record_trivia(self, correct) -- increment __trivia_asked always, increment __trivia_correct only if correct is True

If Something Breaks

Common failure patterns and where they came from:

Symptom Where to Look
ImportError: cannot import name 'Player' from 'player' when running lab-5-2.py Check CP1 (File Setup & Constructor) — verify player.py exists in the same folder as lab-5-2.py and that the class is spelled Player (capital P).
AttributeError: 'Player' object has no attribute '_Player__name' Check CP1 (File Setup & Constructor) — the constructor must assign self.__name = name with exactly two leading underscores.
AttributeError: 'Player' object has no attribute 'get_current_city' (or any other getter method) Check CP2 (Getter Methods) — the missing method name in the error message tells you which getter is missing. lab-5-2.py calls each one by name; if the name doesn't match, the call fails.
travel_to() updates miles but route shows only the destination, not the path Check CP3 (State-Updating Methods) — travel_to must .append() the new city to self.__visited. If it overwrites instead of appends, you lose history.
get_unique_city_count() returns the wrong number (too high or duplicates) Check CP3 (State-Updating Methods) — travel_to must .add() to a set, not .append() to a list. Verify self.__unique_cities is initialized as set() in __init__.
get_trivia_accuracy() crashes with ZeroDivisionError on a fresh player Check CP3 (State-Updating Methods) — when trivia_asked is 0, return '0.0%' (or whatever the spec says) without performing the division. Guard the calculation.
get_route_string() returns the cities but separator is wrong (commas, dashes, no spaces, etc.) Check CP4 (Display Methods) — the separator is exactly ' -> ' (space, hyphen, greater-than, space). Use ' -> '.join(self.__visited).
print(player) shows <player.Player object at 0x...> instead of a summary Check CP4 (Display Methods, __str__) — __str__ must return a string, not print one. If you wrote print(...) instead of return ..., the method returns None and Python falls back to the default repr.
test.py hangs or asks for input when run Check that you haven't added any input() calls to player.py. Player methods do not prompt the user — lab-5-2.py handles all I/O. player.py is data and methods only.
