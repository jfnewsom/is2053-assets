Overview

Last week, you created a City class that knows its own name, connections, and type. But a road trip needs a traveler. This week, you'll create a Game class that tracks the journey — current location, total miles, cities visited. Then you'll make City and Game objects work together. The City displays destinations; the Game tracks your journey. Two classes, one program.

"In Module 4, you had current_city and total_miles as loose variables floating around. Now the Game object owns that state. Ask where you are: game.get_current_city(). Move forward: game.travel_to(...). The object manages itself."

class Game:

    # --- Constructor ---
    __init__(self, name, starting_city)       stores 4 private attributes
        self.__name            = name
        self.__current_city    = starting_city
        self.__total_miles     = 0
        self.__cities_visited  = [starting_city]   start with starting city!

    # --- Getter Methods ---   [CP1]
    get_name(self)                returns self.__name
    get_current_city(self)        returns self.__current_city
    get_total_miles(self)         returns self.__total_miles
    get_cities_visited(self)      returns self.__cities_visited.copy()

    # --- Travel Methods ---   [CP2]
    travel_to(self, city_name, distance)      updates city, miles, visited list

    # --- Display Methods ---   [CP2]
    display_stats(self)           prints team, city, miles, cities count
    __str__(self)                 returns 'Maria in San Antonio'

Why This Matters

Professional games, apps, and systems use a "state object" that carries all the information about what's happening. Instead of scattering variables everywhere, the Game object owns its state. This pattern makes code cleaner, debugging easier, and save/load features possible.

Step-by-Step Instructions
  1. Add a comment # --- Travel Methods ---
  2. Define travel_to(self, city_name, distance):
def travel_to(self, city_name, distance):
    """
    Travel to a new city.

    Args:
        city_name (str): Name of destination city
        distance (int): Distance in miles
    """
    self.__current_city = city_name
    self.__total_miles = self.__total_miles + distance
    self.__cities_visited.append(city_name)
  1. Add a comment # --- Display Methods ---
  2. Define display_stats(self) that prints game statistics:
def display_stats(self):
    """Display game statistics."""
    print(f'Team: {self.__name}')
    print(f'Current City: {self.__current_city}')
    print(f'Total Miles: {self.__total_miles}')
    print(f'Cities Visited: {len(self.__cities_visited)}')
  1. Define __str__(self) that returns a short string representation:
def __str__(self):
    """Return string representation of game."""
    return f'{self.__name} in {self.__current_city}'

Step-by-Step Instructions
  1. Replace your test main() with the real game loop
  2. Start with a title banner, create cities, prompt for team name, and create a Game object
  3. Build the game loop using this structure:
playing = True

while playing:
    # Get current city OBJECT from dictionary
    current_city = cities[game.get_current_city()]

    # Display current location
    print()
    print('-' * 40)
    print(f'You are in: {current_city}')
    print(f'"{current_city.get_tagline()}"')
    print('-' * 40)

    # Check for win
    if current_city.is_win_city():
        print()
        print('YOU REACHED THE BEACH!')
        playing = False
    elif current_city.is_dead_end():
        # Send back to San Antonio
        print()
        print('This is a dead end! No beach here.')
        print('Returning to San Antonio...')
        game.travel_to('San Antonio', 0)
    else:
        # Show destinations and get choice
        current_city.display_destinations()
        num_destinations = len(current_city.get_destination_names())
        choice = get_valid_int(f'Choose (1-{num_destinations}): ', 1, num_destinations)
        # ... convert choice to city name, get distance, travel
  1. In the else branch, use get_valid_int() from your function library — no crashes on bad input!
  2. After the loop ends, display final stats using game.display_stats() and show the full route
