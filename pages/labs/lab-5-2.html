Game Class Foundation

Create game.py with the Game class, __init__ constructor, and getter methods.

A Game class that stores journey state: team name, current city, total miles traveled, and a list of all cities visited.

Professional games, apps, and systems use a "state object" that carries all the information about what's happening. Instead of scattering variables everywhere, the Game object owns its state. This pattern makes code cleaner, debugging easier, and save/load features possible.

"When I first learned OOP, the lightbulb moment was realizing that the Game object carries its own history. You don't need global variables to track where you've been—the game knows. Ask it: game.get_cities_visited(). It remembers for you."

Travel & Display Methods

Add methods that modify game state and display information.

The travel_to() method that updates the game state—new city, adds miles, and records the visit. Plus display and __str__ methods.

  1. Add a comment # --- Travel Methods ---
  2. Define travel_to(self, city_name, distance): 
    def travel_to(self, city_name, distance):
    """
    Travel to a new city.

    Args:
        city_name (str): Name of destination city
        distance (int): Distance in miles
    """
    self.__current_city = city_name
    self.__total_miles = self.__total_miles + distance
    self.__cities_visited.append(city_name)
  3. Add a comment # --- Display Methods ---
  4. Define display_stats(self) that prints game statistics: 
    def display_stats(self):
    """Display game statistics."""
    print(f'Team: {self.__name}')
    print(f'Current City: {self.__current_city}')
    print(f'Total Miles: {self.__total_miles}')
    print(f'Cities Visited: {len(self.__cities_visited)}')
  5. Define __str__(self) that returns a short string representation: 
    def __str__(self):
    """Return string representation of game."""
    return f'{self.__name} in {self.__current_city}'

  2. Start with title banner, create cities, get team name, create Game object
  3. Build the game loop: 
    playing = True

while playing:
    # Get current city OBJECT from dictionary
    current_city = cities[game.get_current_city()]
    
    # Display current location
    print()
    print('-' * 40)
    print(f'You are in: {current_city}')
    print(f'"{current_city.get_tagline()}"')
    print('-' * 40)
    
    # Check for win
    if current_city.is_win_city():
        print()
        print('YOU REACHED THE BEACH!')
        playing = False
    elif current_city.is_dead_end():
        # Handle dead end - send back to San Antonio
        print()
        print('This is a dead end! No beach here.')
        print('Returning to San Antonio...')
        game.travel_to('San Antonio', 0)
    else:
        # Show destinations and get choice
        current_city.display_destinations()
        num_destinations = len(current_city.get_destination_names())
        choice = get_valid_int(f'Choose (1-{num_destinations}): ', 1, num_destinations)
        # ... (convert choice to city name, get distance, travel)
