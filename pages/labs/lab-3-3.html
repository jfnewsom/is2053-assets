Add HAZARD_NAMES List

Replace the if/elif chain for hazard selection with a list and random indexing.

A constant list of hazard names and a function that picks one randomly using list indexing.

Replacing if/elif chains with lists and random indexing is how games scale. Adding 10 more hazards to a list takes seconds; adding 10 more elif branches is tedious and error-prone. This pattern is used in every game for inventory items, enemy types, loot tables, and more.

  1. Near the top of your file (with other constants), add this HAZARD_NAMES list:
HAZARD_NAMES = [
    "Buc-ee's appears!",
    "Armadillo crossing!",
    "Speed trap!",
    "Check engine light!",
    "Whataburger cravings!",
    "Toll road confusion!"
]
  1. Create a get_random_hazard() function with no parameters
  2. Use random.randint(0, len(HAZARD_NAMES) - 1) to get a random index
  3. Return HAZARD_NAMES[index]
  4. Replace the old if/elif hazard chain with a call to this function

BookEx 7.2.2: Shows creating lists with initial values. BookEx 7.5.5: Demonstrates len() for list size—use it to calculate valid index range.

Mistake Why It Fails
Using randint(1, 6) List indices start at 0, so index 6 doesn't exist in a 6-item list
Hardcoding randint(0, 5) If you add more hazards later, you'll get wrong results. Use len() - 1
Forgetting to return the hazard Function returns None, you get no hazard displayed

Sorted Hazards Display

Copy and sort the hazards list for alphabetical display in your trip report.

A sorted copy of your hazards list that displays alphabetically (A-Z) in the trip report.

  1. Find your trip report section (after the game loop ends)
  2. Before displaying hazards, create a copy: sorted_hazards = hazards_list.copy()
  3. Sort the copy: sorted_hazards.sort()
  4. Display the sorted list (either with your new function or manually)

BookEx 7.5.4: Shows the sort() method—it modifies the list in place. That's why we copy first!

Mistake Why It Fails
Using sorted_hazards = hazards_list Both variables point to same list—sorting one sorts both!
Calling hazards_list.sort() directly Destroys original order—can't show hazards in occurrence order elsewhere
Using sorted_hazards = sorted_hazards.sort() sort() returns None, not the sorted list—you'll lose your data

Step 1: Add this temporary test code to verify copy/sort behavior:

print("Sorted:", sorted_test)

Sorted: ["Armadillo crossing!", "Buc-ee's appears!", "Speed trap!"]

Unique Cities Count

Count distinct cities visited using the in operator.

A list of unique cities (no duplicates) and a count to display in your trip report.

  1. In your trip report section, create an empty list: unique_cities = []
  2. Loop through visited_cities with a for loop
  3. For each city, check: if city not in unique_cities:
  4. If not found, append it: unique_cities.append(city)
  5. Display the count using len(unique_cities)

BookEx 7.4: Demonstrates the in operator to check if an item exists in a list. Combine with not to check if absent.

Mistake Why It Fails
Using if city in unique_cities: Logic is backwards—you only add if it's NOT already there
Using len(visited_cities) for unique count That's total visits, not unique cities—includes duplicates
Forgetting to initialize unique_cities = [] NameError: unique_cities is not defined

Step 1: Add this temporary test code to verify the pattern:

# TEMPORARY TEST - DELETE BEFORE SUBMITTING
test_route = ['San Antonio', 'Austin', 'San Antonio', 'Houston', 'Austin']
unique = []
for city in test_route:
    if city not in unique:
        unique.append(city)
print(f"Route: {test_route}")
print(f"Unique: {unique}")
print(f"Unique count: {len(unique)}")

Step 2: Run your program. You should see:

Route: ['San Antonio', 'Austin', 'San Antonio', 'Houston', 'Austin']
Unique: ['San Antonio', 'Austin', 'Houston']
Unique count: 3

Step 3: Verify these behaviors:

  • 5 visits in route, but only 3 unique cities
  • Duplicates filtered — San Antonio and Austin appear once each
  • Order preserved — first occurrence order maintained

Step 4: Play your game, backtrack to some cities, and verify the unique count is correct.

  • Trip report shows "Unique cities visited: X"
  • Count excludes duplicate visits
  • Uses the not in pattern
