Program 8-1: count_Ts.py

Program 8-1 — Iterating Through String Characters
This program counts how many times the letter T appears in a string. The key pattern: a for loop over a string gives you one character per iteration.

What You're Building
  1. Initialize a counter variable count = 0 before the loop.
  2. Get a string from the user with input().
  3. Write a for loop that iterates through each character in the string. The loop variable takes one character at a time.
  4. Inside the loop, check if the character equals 'T' or 't'. If it does, increment the counter.
  5. After the loop, print the total count using an f-string.
Why This Matters

Strings behave like sequences in Python. The same for loop that walks a list walks a string. This is the foundation for understanding why .upper() and testing methods work on every character, doing exactly this iteration internally.

Tips & Pitfalls
Mistake What Goes Wrong Fix
Only checking for 'T' uppercase Misses lowercase 't' Use 'or' to check both: if ch == 'T' or ch == 't'
Initializing count inside the loop Counter resets to 0 on every iteration Initialize count = 0 before the loop starts

Program 8-2: concatenate.py

Program 8-2 — String Concatenation
This short program demonstrates that strings can be joined with + and that reassigning a string variable creates a new string, not a modified version of the original.

What You're Building
  1. Create a variable name = 'Carmen' and print it.
  2. Concatenate ' Brown' to the name using + and assign the result back to name.
  3. Print name again to show the updated value.
Why This Matters

Strings are immutable in Python. When you write name = name + ' Brown', Python creates a brand new string and assigns it to name. Nothing about the original 'Carmen' string changes. This is why string methods like .upper() must be assigned back: text = text.upper(). Otherwise the result is discarded.

Program 8-5: string_test.py

Program 8-5 — String Testing Methods
This program demonstrates six methods that test the contents of a string and return True or False. These are the building blocks of input validation.

What You're Building
  1. Get a string from the user.
  2. Test it with .isalnum() and print a message if True.
  3. Test it with .isdigit() and print a message if True.
  4. Test it with .isalpha() and print a message if True.
  5. Test it with .isspace() and print a message if True.
  6. Test it with .islower() and print a message if True.
  7. Test it with .isupper() and print a message if True.
  8. Run the program several times with different inputs: try a number, a word, all caps, all spaces, and a mix.
Why This Matters

These testing methods are what make robust input validation possible without try/except. If you need to know whether a user typed a number before calling int(), .isdigit() is your first check. Note that multiple tests can be True at once. For example, 'HELLO' passes both .isalpha() and .isupper().

Tips & Pitfalls
Method True when Watch out for
.isdigit() All characters are digits (0–9) Fails on '3.14' because the dot is not a digit
.isalpha() All characters are letters Fails on 'hello world' because the space is not a letter
.islower() All cased characters are lowercase True for 'hello123' since digits do not count as cased
.isupper() All cased characters are uppercase True for 'HELLO123' for the same reason

Program 8-9: string_split.py

Program 8-9 — Basic String Splitting
This program demonstrates .split() with no arguments, which splits a string on any whitespace and returns a list of words.

What You're Building
  1. Create a string variable: my_string = 'One two three four'.
  2. Call my_string.split() and store the result in word_list.
  3. Print word_list to see the resulting list.
Why This Matters

.split() turns a string into a list. Once you have a list, you can loop through it, index into it, or pass it to set() to remove duplicates. This is the pattern behind reading CSV files. Each line is a string, .split(',') gives you individual values.

Program 8-10: split_date.py

Program 8-10 — Split with a Delimiter
This program demonstrates .split('/') to parse a date string in MM/DD/YYYY format into its three components.

What You're Building
  1. Create a date string: date_string = '11/26/2019'.
  2. Call date_string.split('/') and store the result in date_list.
  3. Access the month, day, and year by index: date_list[0], date_list[1], date_list[2].
  4. Print the three components in a labeled format.
Why This Matters

Passing a delimiter character to .split() gives you precise control over where the string breaks. The delimiter itself is consumed and does not appear in the resulting list. This is the standard pattern for parsing any structured string: dates, file paths, CSV records, or user input formatted with separators.

Pro Tip

Try chaining strip and split together: line.strip().split(','). When reading lines from a file, .strip() removes the trailing newline before .split(',') parses the values. Without the strip, the last value in each line will have a hidden \n attached.
