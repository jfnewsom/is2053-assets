Inheritance
Chapter 11  —  4 Programs

Chapter 11 builds directly on Chapter 10. You already know how to define classes — now you'll learn how one class can inherit everything another class has, then add or override behavior of its own. Type every program by hand. These patterns are exactly what you'll use in the Lab 5 assignments.

Learning Objectives
  • Define a superclass (base class) with attributes and methods
  • Define a subclass using class SubClass(SuperClass): syntax
  • Call the superclass constructor from a subclass using SuperClass.__init__(self, ...)
  • Override a superclass method in a subclass
  • Use polymorphism to write a single function that works with any subclass
Key Concepts

Chapter 11 covers inheritance syntax, constructor chaining, method overriding, the "is-a" relationship, and polymorphism. Two of the four programs are merged from multiple textbook listings into a single file.

What You're Building

A single file containing four classes: the Automobile superclass (with make, model, mileage, and price attributes) and three subclasses — Car (adds doors), Truck (adds drive_type), and SUV (adds pass_cap). Each subclass calls the superclass constructor and adds its own attribute on top. This demonstrates the "is-a" relationship in code.

Step-by-Step Instructions
  1. Open your textbook to Programs 11-1 and 11-2.
  2. Create a new file named vehicles.py.
  3. Type Program 11-1 first (the Automobile class with all getters and setters).
  4. Then, in the same file, type Program 11-2 (the Car, Truck, and SUV subclasses).
  5. Save the file — it will be imported by Checkpoint 2.
lightbulb
Superclass Constructor Must Be Called Explicitly

Each subclass constructor must call Automobile.__init__(self, make, model, mileage, price) to initialize the inherited attributes. If you skip this call, the superclass attributes won't exist and you'll get AttributeErrors.

What You're Building

A demonstration program that imports vehicles.py and creates Car, Truck, and SUV objects. It calls getter methods on each to display their attributes — including the inherited ones from Automobile. This is where you see that subclass objects can call methods they inherited without those methods being redefined.

Step-by-Step Instructions
  1. Open your textbook to Program 11-3.
  2. Create a new file named car_demo.py in the same folder as vehicles.py.
  3. Type the full program including the import and all object creation and display steps.
  4. Run it and confirm that the attributes for each vehicle type print correctly.
lightbulb
Polymorphism: One Function, Many Types

The power of polymorphism is that the same function works with any subclass of Mammal, even ones that haven't been written yet. As long as the subclass has show_species() and make_sound(), the function will work.
