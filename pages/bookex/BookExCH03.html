Decision Structures and Boolean Logic
Chapter 3  —  8 Programs

Work through each Gaddis program in order. Chapter 3 is where Python starts making decisions. Type every program by hand and test each one with multiple inputs — especially values right at the boundary of each condition.

Learning Objectives
  • Use a simple if statement to run code only when a condition is true
  • Use if-else to handle two possible outcomes
  • Compare strings to determine alphabetical order
  • Use nested if statements to check multiple conditions in sequence
  • Use if-elif-else to evaluate a series of grade thresholds
  • Combine conditions with the and and or logical operators
Key Concepts

Chapter 3 covers the if statement, if-else, nested decision structures, if-elif-else, comparison operators (>, <, >=, <=, ==, !=), and logical operators (and, or, not).

test_average.py
Program 3-1 (6th Ed) • Program 3-1 (5th Ed)
What You're Building

A program that collects three test scores, calculates the average, and uses a simple if statement to print a congratulations message only when the average meets or exceeds a named constant HIGH_SCORE. If the condition is false, nothing extra happens.

Step-by-Step Instructions
  1. Open your textbook to Program 3-1.
  2. Create a new file named test_average.py.
  3. Type the program exactly as shown, including the HIGH_SCORE constant.
  4. Run it once with scores that average below 95 — confirm no congratulations appear.
  5. Run it again with scores that average at or above 95 — confirm the message appears.
Test Both Paths

Any time you write an if statement, test it at least twice — once where the condition is true and once where it is false. Testing only one path is how bugs hide until a grader finds them.

auto_repair_payroll.py
Program 3-2 (6th Ed) • Program 3-2 (5th Ed)
What You're Building

A payroll program that calculates gross pay differently depending on whether an employee worked overtime. If hours exceed BASE_HOURS (40), overtime pay is calculated at 1.5x the hourly rate. Otherwise, gross pay is simply hours times the pay rate.

Step-by-Step Instructions
  1. Open your textbook to Program 3-2.
  2. Create a new file named auto_repair_payroll.py.
  3. Type the code exactly, including the BASE_HOURS and OT_MULTIPLIER constants.
  4. Run it with 40 hours to confirm straight pay.
  5. Run it again with 45 hours to confirm overtime pay is calculated correctly.
Both Branches Must Calculate gross_pay

The print() statement after the if-else uses gross_pay. If one branch of your if-else does not assign a value to gross_pay, Python will raise a NameError when it tries to print it.

password.py
Program 3-3 (6th Ed) • Program 3-3 (5th Ed)
What You're Building

A simple login program that compares the user's input to a hardcoded password string using the == operator. If the strings match exactly, access is granted. If not, access is denied. Short program, but it introduces the critical difference between = (assignment) and == (comparison).

Step-by-Step Instructions
  1. Open your textbook to Program 3-3.
  2. Create a new file named password.py.
  3. Type the code exactly as shown.
  4. Run it with the correct password and confirm access is granted.
  5. Run it with the wrong password (or wrong capitalization) and confirm denial.
String Comparison Is Case-Sensitive

'Prospero' and 'prospero' are not equal. If the textbook uses lowercase, your check must match exactly. This is the same behavior CodeGrade uses when comparing your output.

sort_names.py
Program 3-4 (6th Ed) • Program 3-4 (5th Ed)
What You're Building

A program that collects two names from the user (last name first) and prints them in alphabetical order. It uses the < operator to compare the two strings and determine which comes first.

Step-by-Step Instructions
  1. Open your textbook to Program 3-4.
  2. Create a new file named sort_names.py.
  3. Type the code exactly as shown.
  4. Run it and enter two names in the opposite of alphabetical order.
  5. Confirm the output prints them in alphabetical order.
Python Compares Strings Character by Character

Python uses Unicode values to compare strings. Uppercase letters have lower values than lowercase, so 'Smith' would sort before 'anderson' in a raw comparison. Enter names consistently (last name first, same case) as the book shows.

loan_qualifier.py
Program 3-5 (6th Ed) • Program 3-5 (5th Ed)
What You're Building

A loan eligibility program that checks two requirements using nested if statements — a minimum salary and a minimum number of years on the job. If the salary check fails, you get a salary-specific message. If salary passes but years fail, you get a years-specific message. Specific feedback for each failure is the point.

Step-by-Step Instructions
  1. Open your textbook to Program 3-5.
  2. Create a new file named loan_qualifier.py.
  3. Type the code exactly, including the MIN_SALARY and MIN_YEARS constants.
  4. Test with salary too low — confirm the salary message appears.
  5. Test with salary OK but years too low — confirm the years message appears.
  6. Test with both requirements met — confirm approval.
Three Test Cases Required

This program has three possible outcomes. You need at least three test runs to verify all paths work: fail salary, pass salary but fail years, and pass both.

grader.py
Program 3-6 (6th Ed) • Program 3-6 (5th Ed)
What You're Building

A grade calculator that takes a numeric score and returns the corresponding letter grade (A through F) using nested if-else statements. Each else branch contains another if, creating a chain that evaluates thresholds in order from highest to lowest.

Step-by-Step Instructions
  1. Open your textbook to Program 3-6.
  2. Create a new file named grader.py.
  3. Type the nested if-else structure exactly as shown — indentation is critical here.
  4. Run it with scores of 95, 85, 75, 65, and 55 to verify all five branches.
Indentation Errors Are the #1 Mistake Here

Each nested level adds 4 more spaces. If your indentation is off by even one space, Python will raise an IndentationError or execute the wrong branch. Count your spaces carefully.

Program 3-7 (6th Ed) • Program 3-7 (5th Ed)
What You're Building

The same loan qualifier scenario as Checkpoint 5, but rewritten to use a single if statement with the and operator instead of nested ifs. If both conditions are true, the loan is approved. If either fails, it is denied. Notice how much simpler the code looks when and replaces nesting.

Step-by-Step Instructions
  1. Open your textbook to Program 3-7.
  2. Create a new file named loan_qualifier2.py.
  3. Type the code, noting the single if with and.
  4. Run it with three test cases: fail salary only, fail years only, and pass both.
  5. Note that this version gives a generic denial message rather than a specific one.
and Requires ALL Conditions to Be True

With and, if even one condition is false, the entire expression is false. Compare this to loan_qualifier.py — same logic, but and is cleaner when you don't need to know which condition failed.

What You're Building

The same loan scenario one more time, now using or. The loan is approved if the applicant meets either the salary requirement OR the years requirement — not necessarily both. This is a deliberately more lenient version of the qualifier, designed to show how or differs from and.

Step-by-Step Instructions
  1. Open your textbook to Program 3-8.
  2. Create a new file named loan_qualifier3.py.
  3. Type the code, noting the switch from and to or.
  4. Run it: meet only salary — should approve. Meet only years — should approve. Meet neither — should deny.
or Requires Only ONE Condition to Be True

With or, the expression is true if at least one condition is true. It is only false when ALL conditions are false. Run the same three test cases you used for loan_qualifier2.py and compare the results.
