Classes and Object-Oriented Programming
Chapter 10  —  4 Programs

Chapter 10 introduces the biggest shift in thinking you will encounter in this course. Until now, your programs have been collections of functions and variables. Classes change that. They let you bundle data and the functions that work on that data into a single unit called an object. Programs 10-9 and 10-10 build a BankAccount class with a special __str__ method. Programs 10-12 and 10-13 build a CellPhone class that demonstrates encapsulation through private attributes and getter/setter methods. Type each one carefully. The indentation and double-underscore naming conventions are not optional.

Learning Objectives
  • Define a class with private attributes using the double-underscore naming convention
  • Write an __init__ constructor method that initializes object attributes
  • Write getter and setter methods to access and modify private attributes
  • Implement the __str__ special method for custom string representation
  • Create object instances and call methods using dot notation

Program 10-9: bankaccount2.py

Program 10-9 — BankAccount Class with __str__
Define a BankAccount class that manages a balance. This file contains only the class definition, with no main() function and no input() calls.

What You're Building
  1. Define class BankAccount:. All the methods below go inside this class, indented one level.
  2. Write __init__(self, bal): store the balance in a private attribute self.__balance = bal.
  3. Write deposit(self, amount): add the amount to self.__balance using +=.
  4. Write withdraw(self, amount): check if self.__balance >= amount. If so, subtract it. If not, print an insufficient funds error message.
  5. Write get_balance(self): return self.__balance.
  6. Write __str__(self): return an f-string formatted as 'The balance is $X,XXX.XX' using the :,.2f format spec on self.__balance.
Why This Matters

The double underscore prefix (__balance) makes the attribute private and it cannot be accessed directly from outside the class. This is encapsulation: the class controls how its data is read and modified. The __str__ method is Python's convention for defining what an object looks like when printed. When you write print(savings), Python calls savings.__str__() automatically.

Tips & Pitfalls
Mistake What Goes Wrong Fix
Using single underscore _balance instead of __balance Attribute is still technically accessible from outside the class Use double underscore __balance for true name mangling
Forgetting self in method signatures TypeError: missing positional argument Every method must have self as its first parameter
Writing balance += amount instead of self.__balance += amount Local variable: balance disappears when method ends Always use self.__ to reference instance attributes

Program 10-12: cellphone.py

Program 10-12 — CellPhone Class with Getters and Setters
Define a CellPhone class with three private attributes and a full set of getter and setter methods. This file contains only the class definition.

What You're Building
  1. Define class CellPhone:.
  2. Write __init__(self, manufact, model, price): store the three parameters in private attributes self.__manufact, self.__model, and self.__retail_price.
  3. Write setter methods: set_manufact(self, manufact), set_model(self, model), and set_retail_price(self, price). Each one assigns its parameter to the corresponding private attribute.
  4. Write getter methods: get_manufact(self), get_model(self), and get_retail_price(self). Each one returns its corresponding private attribute.
Why This Matters

Getters and setters are the standard pattern for controlled attribute access. The class decides what validation (if any) happens when you set a value. In this program they are simple pass-throughs, but in a real system set_retail_price() might reject negative values. The class can enforce that rule in one place instead of everywhere the price is used.

Program 10-13 — Testing the CellPhone Class
This program imports cellphone, creates a CellPhone object, and reads its attributes back using getter methods.

What You're Building
  1. Import cellphone.
  2. Write a main() function. Get manufacturer, model, and retail price from the user.
  3. Create an object: phone = cellphone.CellPhone(man, mod, retail).
  4. Display the stored data by calling the three getter methods: phone.get_manufact(), phone.get_model(), and phone.get_retail_price(). Format the price with $ and two decimal places.
  5. Add the if __name__ == '__main__': guard and call main().
  6. Test it: try accessing phone.__retail_price directly and observe the AttributeError. This is encapsulation working as intended.
Why This Matters

The last test step is the point. When you try phone.__retail_price directly, Python raises an AttributeError. The double-underscore prefix triggers name mangling. Python renames the attribute internally to _CellPhone__retail_price so it cannot be accessed from outside the class with its original name. This is the enforcement mechanism behind encapsulation.
