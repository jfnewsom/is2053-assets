Classes and Object-Oriented Programming
Chapter 10  —  4 Programs

Chapter 10 is a turning point. You're moving from procedural programming to object-oriented programming. Type every program by hand — these patterns (class definition, __init__, getters, setters, __str__) will appear in every Lab assignment from here on.

Learning Objectives
  • Define a class with private attributes using the double-underscore prefix
  • Write an __init__ constructor that initializes attributes from parameters
  • Write getter and setter methods for controlled attribute access
  • Add a __str__ method so objects can be printed cleanly
  • Import a class from another module and create object instances
  • Call methods on objects using dot notation
Key Concepts

Chapter 10 covers class definition, the __init__ constructor, private attributes (self.__attribute), getter and setter methods, the __str__ special method, and importing classes from modules.

What You're Building

The BankAccount class — a blueprint for a bank account object. The class stores a private balance, provides deposit() and withdraw() methods to modify it, a get_balance() getter, and a __str__ method so the object can be printed directly. This is a class definition file only — it does not run on its own.

Step-by-Step Instructions
  1. Open your textbook to Program 10-9.
  2. Create a new file named bankaccount2.py.
  3. Type the full class definition including all methods.
  4. Save the file — you will not run it directly. It will be imported by Checkpoint 2.
__str__ Is Called Automatically by print()

When you write print(savings) in Checkpoint 2, Python automatically calls savings.__str__() and prints what it returns. You never call __str__ directly — Python handles it behind the scenes.

What You're Building

The CellPhone class with three private attributes (__manufact, __model, __retail_price), an __init__ constructor, and a full set of getter and setter methods for each attribute. This is the standard pattern you will use in every Lab that involves classes.

Step-by-Step Instructions
  1. Open your textbook to Program 10-12.
  2. Create a new file named cellphone.py.
  3. Type the full class definition including all six getter and setter methods.
  4. Save the file — it will be imported by Checkpoint 4.
Every Method Must Have self as Its First Parameter

Every method inside a class — including __init__, getters, and setters — must have self as its first parameter. You do not pass self when calling the method; Python handles that automatically.

What You're Building

A test program that imports cellphone.py, collects phone data from the user, creates a CellPhone object, and uses the getter methods to display what was entered. This is the standard two-file OOP pattern: one file defines the class, one file uses it.

Step-by-Step Instructions
  1. Open your textbook to Program 10-13.
  2. Create a new file named cell_phone_test.py in the same folder as cellphone.py.
  3. Type the full program including the import statement, user input, object creation, and display.
  4. Run cell_phone_test.py and enter manufacturer, model, and price when prompted.
  5. Confirm all three values are echoed back correctly.
Object Creation Syntax

To create an object: phone = cellphone.CellPhone(man, mod, retail). The module name comes first, then a dot, then the class name, then the constructor arguments. The arguments are passed directly to __init__.
