Learning Objectives
  • Use a sentinel-controlled while loop to repeat until the user says stop
  • Use a condition-controlled while loop that runs until a value meets a threshold
  • Use a for loop with a list to iterate through a fixed sequence
  • Use range() to repeat code a specific number of times
  • Use range(start, end) and range(start, end, step) for custom sequences
  • Build an accumulator variable to calculate a running total
  • Use an inner while loop for input validation
  • Use nested for loops to process multi-dimensional data
Key Concepts

Chapter 4 covers while loops (sentinel-controlled and condition-controlled), for loops, the range() function with one, two, and three arguments, accumulators, input validation loops, and nested loops.

commission.py

What You're Building

A commission calculator that uses a sentinel-controlled while loop. The user enters sales data and gets a commission amount, then decides whether to run the loop again by entering 'y'. The loop stops when anything other than 'y' is entered.

Step-by-Step Instructions
  1. Open your textbook to Program 4-1.
  2. Create a new file named commission.py.
  3. Type the program exactly as shown, including the keep_going initialization before the loop.
  4. Run it and process at least two commissions, then exit.
  5. Run it again and enter something other than 'y' after the first commission to verify the loop stops.
Initialize the Sentinel Variable Before the Loop

The while loop checks keep_going before the first iteration. If the variable doesn't exist yet, Python raises a NameError. Always initialize your loop control variable before the while statement.

What You're Building

A temperature monitor that loops as long as the input temperature exceeds a MAX_TEMP constant. The user is prompted to adjust the thermostat and re-enter the temperature until it falls within the acceptable range.

Step-by-Step Instructions
  1. Open your textbook to Program 4-2.

Update the Variable Inside the Loop

The while loop checks temperature on every iteration. If you never update it inside the loop, the condition never changes and the loop runs forever. Press Ctrl+C (Windows) or Command+Period (Mac) to stop an infinite loop.

simple_loop4.py
Program 4-12 (6th Ed) • Program 4-7 (5th Ed)
What You're Building

A for loop that uses range(5) to print 'Hello world!' exactly five times. This is the standard introduction to range() — the most common way to control how many times a loop runs.

Step-by-Step Instructions
  1. Open your textbook to Program 4-12 (6th Ed) or Program 4-7 (5th Ed).
  2. Create a new file named simple_loop4.py.
  3. Type the program exactly as shown.
  4. Run it and confirm exactly five lines of output appear.
range(5) Generates 0 Through 4

range(5) produces 0, 1, 2, 3, 4 — five values starting at zero. The loop variable x gets each of these, but since the print statement doesn't use x, the same message prints five times.

squares.py
Program 4-13 (6th Ed) • Program 4-8 (5th Ed)
What You're Building

A table of numbers 1 through 10 and their squares, formatted with tab stops to create aligned columns. Introduces range(1, 11) and the ** exponentiation operator.

Step-by-Step Instructions
  1. Open your textbook to Program 4-13 (6th Ed) or Program 4-8 (5th Ed).
  2. Create a new file named squares.py.
  3. Type the program including the header lines with \t tab characters.
  4. Run it and confirm the table goes from 1 to 10 with correct squares.
range(1, 11) Not range(1, 10)

range(start, end) goes from start up to but NOT including end. To get numbers 1 through 10, you need range(1, 11). Using range(1, 10) only gives you 1 through 9.

speed_converter.py
Program 4-14 (6th Ed) • Program 4-9 (5th Ed)
What You're Building

A KPH-to-MPH conversion table that uses range(START_SPEED, END_SPEED, INCREMENT) to step from 60 to 130 in increments of 10. This is the three-argument form of range(), and the named constants make the step size easy to read and modify.

Step-by-Step Instructions
  1. Open your textbook to Program 4-14 (6th Ed) or Program 4-9 (5th Ed).
  2. Create a new file named speed_converter.py.
  3. Type the program including all four named constants at the top.
  4. Run it and confirm the table starts at 60 KPH and steps by 10 up to 130 KPH.
Why END_SPEED Is 131, Not 130

range() excludes the end value. If END_SPEED were 130, the last value generated would be 120. Setting it to 131 ensures 130 itself is included in the table.

user_squares2.py
Program 4-16 (6th Ed) • Program 4-11 (5th Ed)
What You're Building

The squares table from Checkpoint 5, but now the user controls the starting and ending numbers. Demonstrates how to use input values directly in range() and why end + 1 is needed to make the range inclusive.

Step-by-Step Instructions
  1. Open your textbook to Program 4-16 (6th Ed) or Program 4-11 (5th Ed).
  2. Create a new file named user_squares2.py.
  3. Type the program as shown.
  4. Run it and enter a start of 3 and an end of 7 — confirm the table shows 3 through 7.
  5. Try entering the same number for both start and end to see what happens.
Use end + 1 to Include the Ending Value

If the user enters 7, range(start, end) stops at 6. Using range(start, end + 1) includes 7 as expected. This off-by-one pattern is one of the most common bugs in Python loops.

sum_numbers.py
Program 4-17 (6th Ed) • Program 4-12 (5th Ed)
What You're Building

A program that asks the user to enter five numbers and calculates their sum using an accumulator variable. The accumulator is initialized to 0.0 before the loop and updated with += inside it. This pattern appears in virtually every Lab assignment.

Step-by-Step Instructions
  1. Open your textbook to Program 4-17 (6th Ed) or Program 4-12 (5th Ed).
  2. Create a new file named sum_numbers.py.
  3. Type the program exactly, noting where total = 0.0 is placed relative to the loop.
  4. Run it and enter five numbers.
  5. Verify the total equals the sum of what you entered.
Initialize the Accumulator BEFORE the Loop

If total = 0.0 is inside the loop, it resets to zero on every iteration and you only ever get the last number entered. The accumulator must be set before the loop starts so it can collect values across all iterations.

What You're Building

A retail price calculator with two levels of while loops. The outer loop lets the user process multiple items. The inner loop validates the wholesale cost — if a negative number is entered, the program displays an error and asks again. Only after a valid cost is entered does the retail calculation proceed.

Step-by-Step Instructions
  1. Open your textbook to Program 4-21 (6th Ed) or Program 4-16 (5th Ed).
  2. Create a new file named retail_with_validation.py.
  3. Type the program carefully, paying attention to both loop levels and their indentation.
  4. Run it and enter a negative number first — confirm the error message appears.
  5. Enter a valid positive number and confirm the retail price calculates correctly.
  6. Test the outer loop by entering 'y' to process a second item.
Input Validation Is a Core Pattern

The inner validation loop — get input, then while it's invalid display an error and re-prompt — is a pattern you will use in almost every Lab assignment. Get comfortable with this structure now.

test_score_averages.py
Program 4-22 (6th Ed) • Program 4-17 (5th Ed)
What You're Building

A grade calculator that uses nested for loops to process multiple students, each with multiple test scores. The outer loop iterates through students; the inner loop collects scores and accumulates the total. The average is calculated and printed after the inner loop completes for each student.

Step-by-Step Instructions
  1. Open your textbook to Program 4-22 (6th Ed) or Program 4-17 (5th Ed).
  2. Create a new file named test_score_averages.py.
  3. Type the program carefully, paying attention to indentation at each loop level.
  4. Run it with 2 students and 3 test scores each as a first test.
  5. Confirm each student's average is correct and that the accumulator resets between students.
Reset the Accumulator Inside the Outer Loop

total = 0.0 must be inside the outer loop but outside the inner loop. Before the outer loop: only resets once. Inside the inner loop: resets after every score. The right place is at the start of each student's processing — indented under the outer for, above the inner for.
