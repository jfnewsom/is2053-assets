Program 9-1: card_dealer.py

Program 9-1 — Card Dealer with Dictionary
This program uses a dictionary as a deck of cards, where each key is a card name (string) and each value is the card's point value (integer). Three functions work together: main() coordinates, create_deck() builds and returns the dictionary, and deal_cards() randomly selects cards and accumulates their values.

What You're Building
  1. Import the random module at the top.
  2. Write create_deck(): build a dictionary with card names as keys and integer values as values. Include all 52 cards across four suits (Spades, Hearts, Clubs, Diamonds). Face cards (Jack, Queen, King) are worth 10. Aces are worth 1. Number cards match their number. Return the dictionary.
  3. Write deal_cards(deck, number): initialize hand_value = 0. If number is greater than len(deck), cap it at len(deck). Use a for loop with range(number). Inside the loop, select a random card with random.choice(list(deck)), print the card name, and add its value to hand_value. After the loop, print the total hand value.
  4. Write main(): call create_deck(), get the number of cards from the user, then call deal_cards().
  5. Add the if __name__ == '__main__': guard and call main().
Why This Matters

The line random.choice(list(deck)) is the key technique. random.choice() requires a sequence, but dictionaries are not sequences. Converting with list(deck) gives you a list of keys, and random.choice() picks one. Then deck[card] looks up the value. This is the same pattern behind the random.choice(HAZARD_KEYS) in your game.

Tips & Pitfalls
Mistake What Goes Wrong Fix
random.choice(deck) without list() TypeError: dict is not a sequence Wrap it: random.choice(list(deck))
Forgetting to return deck from create_deck() main() gets None, deal_cards() crashes Add return deck at the end of create_deck()
Duplicate card names in the dictionary Silently overwrites. Python keeps only the last value Each key must be unique. '10 of Spades' is different from '10 of Hearts'

Program 9-3: sets.py

Program 9-3 — Set Operations
This program creates two sets of student athletes and then demonstrates all four set operations: intersection, union, difference, and symmetric difference. Note: this program has no functions and runs entirely at the module level.

What You're Building
  1. Create two sets using the set() constructor with a list argument: baseball = set(['Jodi', 'Carmen', 'Aida', 'Alicia']) and basketball = set(['Eva', 'Carmen', 'Alicia', 'Sarah']).
  2. Use a for loop to print each member of the baseball set, preceded by a label. Do the same for basketball.
  3. Print students in BOTH sets using baseball.intersection(basketball) in a for loop.
  4. Print students in EITHER set using baseball.union(basketball) in a for loop.
  5. Print baseball-only students using baseball.difference(basketball).
  6. Print basketball-only students using basketball.difference(baseball).
  7. Print students in exactly one sport using baseball.symmetric_difference(basketball).
  8. Run the program. Notice the output order is not alphabetical because sets are unordered.
Why This Matters

The four set operations map to real questions you ask about data: who is in both groups, who is in either group, who is only in this group, who is in exactly one group. These operations are O(n) in Python, faster than looping through lists manually for the same result. When you used unique_cities_visited.add(city) in lab 4-3, you were using sets for their most common purpose: automatic deduplication.

Tips & Pitfalls
Mistake What Goes Wrong Fix
Creating empty set with {} Creates an empty dict, not a set Always use set() for an empty set
Expecting alphabetical output from set iteration Order is not guaranteed Wrap in sorted() if order matters: for name in sorted(my_set):
Confusing .difference() direction baseball.difference(basketball) vs basketball.difference(baseball) give different results Read it as: first_set minus second_set
