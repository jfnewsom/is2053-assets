Lists and Tuples
Chapter 7  —  10 Programs

Work through each Gaddis program in order. Chapter 7 introduces lists — the most important data structure in the course. Type every program by hand. Lists unlock a whole new level of problem-solving, and the patterns here (append, iterate, accumulate, search, modify) show up in every Lab assignment from here on.

All Program Numbers Match Between Editions

Unlike Chapters 4, 5, and 6, all Chapter 7 program numbers are identical between the 5th and 6th Editions. Also note: run Checkpoint 9 (write_number_list.py) before Checkpoint 10 (read_number_list.py) — the second one reads a file the first one creates.

Learning Objectives
  • Create a list with a fixed size using the repetition operator
  • Build a list dynamically using append() in a loop
  • Search a list using the in operator
  • Find an item's index using index() and handle ValueError with try-except
  • Add items at a specific position using insert()
  • Remove items using remove() and handle ValueError with try-except
  • Calculate a list average using both a manual accumulator and sum()
  • Combine functions that build, process, and return lists
  • Write a list to a file by looping and converting each item to a string
  • Read a file into a list using readlines() and convert each element
Key Concepts

Chapter 7 covers list creation, indexing, the in operator, append(), insert(), remove(), index(), sum(), len(), min(), max(), passing lists to functions, returning lists from functions, and file I/O with lists.

sales_list.py
Program 7-1 (6th Ed) • Program 7-1 (5th Ed)
What You're Building

A program that creates a five-element list pre-filled with zeros using the repetition operator ([0] * NUM_DAYS), then uses a for loop with range(len(sales)) to collect a sales value for each index. A second for loop displays all values. This is the standard pattern for working with a list of known size.

Step-by-Step Instructions
  1. Open your textbook to Program 7-1.
  2. Create a new file named sales_list.py.
  3. Type the program as shown, including the NUM_DAYS constant.
  4. Run it and enter five sales values.
  5. Confirm all five values are displayed back correctly.
[0] * 5 Creates a List of Five Zeros

The expression [0] * NUM_DAYS creates a list with NUM_DAYS elements, each initialized to 0. This reserves the space before you fill it. The for loop then replaces each 0 with user input using the index: sales[index] = float(input(...)).

append() Always Adds to the End

Each call to name_list.append(name) adds the new name after all existing items. Starting with an empty list [] and building with append() is the pattern you will use in almost every Lab that collects a variable number of items.

What You're Building

A program that lets the user change an item in a list by searching for it with index(), which returns the item's position. The program gets a replacement value and overwrites the old item at that position. A try-except block catches ValueError if the item isn't found.

Step-by-Step Instructions
  1. Open your textbook to Program 7-4.
  2. Create a new file named index_list.py.
  3. Type the program as shown including the try-except block.
  4. Run it and change 'Burgers' to 'Tacos' — confirm the list updates.
  5. Run it again and try to change 'Sushi' (not in the list) — confirm the error message appears.
index() Raises ValueError If the Item Is Not Found

If the item you're searching for isn't in the list, index() raises ValueError — which crashes your program if unhandled. The try-except block catches that and prints a friendly message instead. Always protect index() calls with try-except or an in check first.

What You're Building

A program that creates a list with three names, displays it, then inserts 'Joe' at position 0 (the very beginning) using insert(0, 'Joe'). All existing items shift one position to the right. The list is displayed again to show the result.

Step-by-Step Instructions

  3. Type the program as shown.

insert(index, item) Shifts Everything Right

insert(0, 'Joe') puts 'Joe' at the front and shifts 'James', 'Kathryn', and 'Bill' to positions 1, 2, and 3. Unlike append(), insert() can place an item at any position. insert(len(list), item) is equivalent to append(item).

A program that calculates the average of a list of scores two different ways: first with a manual accumulator loop (total1), then with the built-in sum() function (total2). Both produce the same result, showing that sum() is a clean shortcut for the accumulator pattern.

sum(), min(), max(), and len() Work on Any Numeric List

sum(scores) replaces a for loop accumulator. len(scores) gives the count. min(scores) and max(scores) find the extremes. You will use all four of these in upcoming Lab assignments.
