time_converter.py
Program 2-17 (6th Ed) • Program 2-17 (5th Ed)
What You're Building

A program that takes a total number of seconds and converts it to hours, remaining minutes, and remaining seconds using integer division (//) and modulo (%). These two operators come up constantly throughout the rest of the semester.

Step-by-Step Instructions
  1. Open your textbook to Program 2-17.
  2. Create a new file named time_converter.py.
  3. Type the code exactly as shown.
  4. Run it using a value like 3661 seconds.
  5. Verify the hours, minutes, and seconds output is correct.
// vs %

// gives you the quotient (how many times it divides evenly). % gives you the remainder. For 3661 seconds: 3661 // 3600 = 1 hour, 3661 % 3600 = 61 remaining seconds.

A program that collects a first and last name from the user, joins them together with a space using the + operator, and prints the full name. String concatenation was the original way to build output before f-strings existed, and it is still useful for simple joins.

You Cannot Concatenate a String and a Number

Using + to join a string and an int raises a TypeError. Use str() to convert the number first, or switch to an f-string.

f_string_no_formatting.py
Program 2-20 (6th Ed) • Program 2-20 (5th Ed)
What You're Building

A program that calculates a monthly payment and displays it using an f-string with no formatting specifier — just the basic f'...{variable}...' syntax. This is the modern, preferred way to build output in Python, and you will use f-strings in every Lab assignment this semester.

Step-by-Step Instructions
  1. Open your textbook to Program 2-20.
  2. Create a new file named f_string_no_formatting.py.
  3. Type the code, paying attention to the f prefix before the opening quote.
  4. Run and verify the variable value appears in the output.
  5. Try changing the variable value to see it update in the output.
Don't Forget the f Prefix

Without the f before the quote, Python prints the literal text {monthly_payment} instead of the value. This is an easy mistake and a very common one.

f_string_rounding.py
Program 2-21 (6th Ed) • Program 2-21 (5th Ed)
What You're Building

The same monthly payment program as Checkpoint 7, but now with :.2f inside the curly braces to round the output to two decimal places. You will use this pattern in almost every Lab assignment this semester to display currency and calculated values cleanly.

Step-by-Step Instructions
  1. Open your textbook to Program 2-21.
  2. Create a new file named f_string_rounding.py.
  3. Type the code, noting the :.2f inside the curly braces.
  4. Run and verify the number is rounded to two decimal places.
  5. Try changing :.2f to :.4f and see what happens.
:.2f Means Two Decimal Places

The colon starts the format spec, .2 sets precision to 2 decimal places, and f means floating-point. The entire spec goes inside the curly braces: {monthly_payment:.2f}.

columns.py
Program 2-23 (6th Ed) • Program 2-23 (5th Ed)
What You're Building

A program that prints six decimal values arranged in two right-aligned columns using the :10.2f format specifier. The 10 reserves a field of 10 characters; the .2f rounds to two decimal places. The result is a clean, aligned table.

Step-by-Step Instructions
  1. Open your textbook to Program 2-23.
  2. Create a new file named columns.py.
  3. Type the code carefully, paying close attention to the format specifiers.
  4. Run the program and verify the numbers form neat aligned columns.
  5. Compare your output to the book to confirm alignment is correct.
Field Width Controls Spacing

The number before the decimal in the format spec (10 in :10.2f) is the field width. All values right-align within that field, which is what creates the column effect.

center_align.py
Program 2-24 (6th Ed) • Program 2-24 (5th Ed)
What You're Building

A program that prints six names, each centered in a 20-character field using the :^20 format specifier. The *** markers on each side let you see exactly how wide the field is and confirm the text is centered.

Step-by-Step Instructions
  1. Open your textbook to Program 2-24.
  2. Create a new file named center_align.py.
  3. Type the example and confirm the output is centered correctly.
  4. Try changing :^20 to :<20 (left) and :>20 (right) to see the difference.
Three Alignment Specifiers

^ centers text, < left-aligns text, and > right-aligns text (the default for numbers). All three go inside the curly braces before the field width number.
