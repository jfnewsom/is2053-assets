  • Define functions using the def keyword and call them from main()

  • Understand that local variables in different functions are independent
  • Pass data into functions using arguments and parameters
  • Use global constants (defined outside all functions) across multiple functions
  • Return values from functions using the return statement
  • Import and use the math and random modules
  • Use the if __name__ == '__main__': guard for reusable modules
Key Concepts

Chapter 5 covers function definition and calling, local variable scope, passing arguments, global constants, return values, importing the math and random modules, and the if __name__ == '__main__' guard.

Checkpoint 1
What You'Re Building

A program with two functions: main() and message(). The main() function calls message(), which prints two lines, then execution returns to main() and continues. This is the simplest demonstration of how function calls transfer and return control.

Step-By-Step Instructions
  1. Open your textbook to Program 5-2.
  2. Create a new file named two_functions.py.
  3. Type the program exactly as shown, including the call to main() at the bottom.
  4. Run it and trace the output to confirm the execution order matches the call sequence.
Define Functions at the Top, Call main() at the Bottom

Python reads all function definitions when the file loads. That's why main() can call message() even though message() is defined after main() — by the time main() runs, Python already knows about message(). Always put the call to main() at the very bottom of the file.

What You'Re Building

A step-by-step instruction program for disassembling a dryer, broken into six functions: startup_message(), step1() through step4(), and main() which calls them all in sequence and pauses for user input between steps. This is the pattern of organizing a program into single-purpose functions.

Step-By-Step Instructions

  2. Create a new file named acme_dryer.py.
  3. Type all six function definitions and the call to main() at the bottom.
  4. Run it and press Enter to step through each section.
  5. Confirm each step's message appears in the correct order.
Each Function Has One Job

Notice that startup_message() only handles the intro, step1() only handles step 1, and so on. main() does nothing except coordinate the others. This is the single-responsibility principle, and it's what makes code easy to read and maintain.

Checkpoint 3
What You'Re Building

A program with two functions — texas() and california() — that each have a local variable named birds. Despite sharing the same name, the two variables are completely independent. This demonstrates that local variables belong to their function and can't be seen or affected by other functions.

Step-By-Step Instructions
  1. Open your textbook to Program 5-5.
  2. Create a new file named birds.py.
  3. Type the program as shown.
  4. Run it and confirm texas() prints 5000 and california() prints 8000.
  5. Notice that changing birds in one function would have zero effect on the other.
Local Variables Only Exist Inside Their Function

The birds variable in texas() and the birds variable in california() are two completely separate variables that happen to share a name. When texas() finishes, its birds disappears. This independence is called scope, and it prevents functions from accidentally interfering with each other.

Checkpoint 4
What You'Re Building

A unit converter that collects cups from the user in main(), then passes the value to cups_to_ounces() as an argument. The function receives it as the parameter cups, calculates the ounces, and prints the result. This is the core argument-passing pattern you will use in every Lab assignment.

Step-By-Step Instructions
  1. Open your textbook to Program 5-7.
  2. Create a new file named cups_to_ounces.py.
  3. Type the program including the intro() function.
  4. Run it and enter a number of cups.
  5. Confirm the correct number of ounces is displayed.
Arguments vs Parameters

When main() calls cups_to_ounces(cups_needed), cups_needed is the argument — the value being passed. When cups_to_ounces(cups) receives it, cups is the parameter — the variable inside the function that holds the value. Same data, different names depending on which side of the call you're describing.

Checkpoint 5
What You'Re Building

A program where main() calls show_sum(12, 45), passing two arguments. The function accepts two parameters, calculates their sum, and returns it. main() stores the returned value in total and prints it. This demonstrates both multiple arguments and the return statement.

Step-By-Step Instructions
  1. Open your textbook to Program 5-8.
  2. Create a new file named multiple_args.py.
  3. Type the program exactly as shown, including the commented-out print inside show_sum.
  4. Run it and confirm the output shows 57.
return Sends Data Back; print Only Displays It

If show_sum used print(result) instead of return result, main() would receive None when it calls show_sum(12, 45) — and total would be None, not 57. Use return when you need the caller to be able to use the value. Use print only for final display.

Checkpoint 6
What You'Re Building

A retirement contribution calculator with a global constant CONTRIBUTION_RATE defined outside all functions. Both show_pay_contrib() and show_bonus_contrib() use it in their calculations without needing it passed as an argument. This demonstrates the correct use of global constants.

Step-By-Step Instructions
  1. Open your textbook to Program 5-17 (6th Ed) or Program 5-15 (5th Ed).
  2. Create a new file named retirement.py.
  3. Type the CONTRIBUTION_RATE constant at the very top, before any function definitions.
  4. Run it and enter a gross pay and bonus amount.
  5. Confirm both contributions are calculated at 5% of the amounts entered.
Global Constants Are Fine; Global Variables Are Not

CONTRIBUTION_RATE is a constant — it never changes. That makes it safe to share across functions. What you want to avoid is modifying a global variable from inside a function, which makes your code unpredictable. Constants (ALL_CAPS) shared for reading: fine. Variables modified from inside functions: avoid.

Checkpoint 7
What You'Re Building

A dice simulator that imports the random module and uses random.randint(MIN, MAX) to generate two random values between 1 and 6. A sentinel loop lets the user keep rolling. This is the standard random module introduction and the pattern used in the game Lab assignments.

Step-By-Step Instructions
  1. Open your textbook to Program 5-21 (6th Ed) or Program 5-19 (5th Ed).
  2. Create a new file named dice.py.
  3. Type the program including the import random statement at the top and the MIN/MAX constants.
  4. Run it and roll the dice several times.
  5. Confirm the values are between 1 and 6 and change with each roll.
random.randint() Is Inclusive on Both Ends

random.randint(1, 6) can return 1, 2, 3, 4, 5, or 6 — both endpoints included. This is different from range(), which excludes the end. The two calls in this program simulate two separate dice rolls, so both values can be different on the same roll.

What You'Re Building

A sale price calculator with three functions: get_regular_price() returns user input, discount(price) returns the discount amount, and main() chains them together: sale_price = reg_price - discount(reg_price). This shows how return values can be used directly in expressions and how functions can call other functions.

Step-By-Step Instructions
  1. Open your textbook to Program 5-24 (6th Ed) or Program 5-22 (5th Ed).
  2. Create a new file named sale_price.py.
  3. Type the program including the DISCOUNT_PERCENTAGE global constant.
  4. Run it and enter a price like 50.00.
  5. Confirm the sale price is 20% less than the regular price.
Return Values Can Be Used Directly in Expressions

sale_price = reg_price - discount(reg_price) calls discount() and subtracts its return value in a single line. You don't have to store the return value in a variable first unless you need to use it more than once.

What You'Re Building

A right triangle hypotenuse calculator that imports the math module and calls math.hypot(a, b). This demonstrates import math and dot notation for module functions. The math module provides pre-built mathematical functions so you don't have to implement them yourself.

Step-By-Step Instructions
  1. Open your textbook to Program 5-28 (6th Ed) or Program 5-26 (5th Ed).
  2. Create a new file named hypotenuse.py.
  3. Type the program with import math at the top.
  4. Run it and enter side lengths of 3 and 4 — the hypotenuse should be 5.0.
  5. Try other values to confirm it works.
Dot Notation for Module Functions

math.hypot(a, b) means: from the math module, call the hypot function. The module name comes first, then a dot, then the function name. This is the same pattern as math.sqrt(), math.floor(), and any other module function.

What You'Re Building

A rectangle calculator with separate area() and perimeter() functions that each accept width and length and return a calculated value. main() calls both and uses their return values directly in print statements. This also introduces the if __name__ == '__main__': guard, which is the standard way to structure reusable Python files.

Step-By-Step Instructions
  1. Open your textbook to Program 5-32 (6th Ed) or Program 5-30 (5th Ed).
  2. Create a new file named rectangle2.py.
  3. Type the program including the if __name__ == '__main__': guard at the bottom instead of a bare main() call.
  4. Run it and enter width and length values.
  5. Confirm the area and perimeter are both calculated correctly.
Why if __name__ == '__main__': Matters

When Python runs a file directly, __name__ equals '__main__'. When the file is imported by another file, __name__ equals the filename instead. The guard ensures main() only runs when you execute the file directly — not when you import it. You will see this pattern in every Lab assignment from here on.
