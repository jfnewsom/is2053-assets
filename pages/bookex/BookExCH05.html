lightbulb
Define Functions at the Top, Call main() at the Bottom

Python reads all function definitions when the file loads. That's why main() can call message() even though message() is defined after it — by the time main() runs, Python already knows about message(). Always put the call to main() at the very bottom of the file.

lightbulb
Arguments vs Parameters

When main() calls cups_to_ounces(cups_needed), cups_needed is the argument — the value being passed. When cups_to_ounces(cups) receives it, cups is the parameter — the variable inside the function that holds it. Same data, different names depending on which side of the call you're describing.

lightbulb
Global Constants Are Fine; Global Variables Are Not

CONTRIBUTION_RATE is a constant — it never changes. That makes it safe to share across functions. What you want to avoid is modifying a global variable from inside a function, which makes code unpredictable. Constants in ALL_CAPS used for reading: fine. Variables modified from inside functions: avoid.

info
Dot Notation for Module Functions

math.hypot(a, b) means: from the math module, call the hypot function. The module name comes first, then a dot, then the function name. This is the same pattern as math.sqrt(), math.floor(), and random.randint().

Program 5-32 (6th Ed) • Program 5-30 (5th Ed)
What You're Building

A rectangle calculator with separate area() and perimeter() functions that each accept width and length and return a calculated value. main() calls both and uses their return values directly in print statements. This also introduces the if __name__ == '__main__': guard — the standard structure for all reusable Python files.

Step-by-Step Instructions
  1. Open your textbook to Program 5-32 (6th Ed) or Program 5-30 (5th Ed).
  2. Create a new file named rectangle2.py.
  3. Type the program using if __name__ == '__main__': at the bottom instead of a bare main() call.
  4. Run it and enter width and length values.
  5. Confirm the area and perimeter are both calculated correctly.
info
Why if __name__ == '__main__': Matters

When Python runs a file directly, __name__ equals '__main__'. When the file is imported by another file, __name__ equals the filename instead. The guard ensures main() only runs when you execute the file directly — not when it's imported. You will use this pattern in every Lab assignment from here on.
